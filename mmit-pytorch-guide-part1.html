Day 1: 基础层

1.1 ReLU — 最简单的激活函数

是什么：f(x) = max(0, x)。负值归零，正值不变。计算极其简单高效。

为什么需要：如果没有非线性激活，多个线性层堆叠 = 一个线性层（矩阵乘法的结合律）。加入 ReLU 后，网络才能学习非线性关系。

ReLU: f(x) = max(0, x) y 3 | · | · 2 | · | · 1 | · | · 0 |· · · · · · · +--+--+--+--+--+--+--+--+--+-→ x -3 -2 -1 0 1 2 3
ReLU 示例
输入: -2.0 -0.5 0.0 1.3 3.0 → ReLU → 0.0 0.0 0.0 1.3 3.0
PyTorch 用法
import torch
import torch.nn as nn
import torch.nn.functional as F

x = torch.tensor([-2.0, -0.5, 0.0, 1.3, 3.0])

# 三种等价写法
y1 = torch.relu(x)            # 函数式
y2 = F.relu(x)                # functional API
y3 = nn.ReLU()(x)             # Module（可放入 nn.Sequential）

激活函数对比：ReLU vs GELU vs SiLU

名称公式特点使用场景
ReLUmax(0, x)简单高效，x=0 处不可导，负半轴梯度为零（"死神经元"）经典 CNN、基础模型
GELUx · Φ(x)平滑近似 ReLU，允许少量负值通过BERT, GPT 系列
SiLU/Swishx · σ(x)平滑、无上界、有下界。LLaMA 的 SwiGLU 中使用LLaMA, mmit 基础模型
激活函数对比（近似图示） y 3 | R G S R=ReLU G=GELU S=SiLU | R G S 2 | R G S | R GS 1 | RGS | GS· 0 |· · · · ·R· · · 注意：SiLU 在 x≈-1 处有 | S G 小的负值（约 -0.28） -1 | S +--+--+--+--+--+--+--+--+-→ x -3 -2 -1 0 1 2 3
mmit 关联：LLaMA 使用 SwiGLU（= SiLU 激活 + 门控线性单元），不直接用 ReLU。但理解 ReLU 是理解所有激活函数的基础。

1.2 Softmax — 把数字变成概率

是什么：softmax(xi) = exp(xi) / Σ exp(xj)。将任意实数向量转换为概率分布（每个值在 0-1 之间，总和为 1）。

为什么需要：分类任务最后需要概率输出；Attention 机制中需要将 score 转为注意力权重。

Softmax 计算示例
logits: 2.0 1.0 0.1 → exp: 7.39 2.72 1.11 → ÷ 11.22 → 0.659 0.242 0.099
概率之和：0.659 + 0.242 + 0.099 = 1.000 ✓

数值稳定性：直接计算 exp(1000) = ∞（溢出）！解决方法：先减去最大值。

数值稳定技巧
1000 1001 999 → 减去 max(1001): -1 0 -2 → exp 安全计算 ✓
数学上等价：softmax(x) = softmax(x - max(x))，PyTorch 内部自动处理
PyTorch 用法
logits = torch.tensor([2.0, 1.0, 0.1])
probs = F.softmax(logits, dim=-1)   # dim=-1 表示沿最后一个维度
# tensor([0.6590, 0.2424, 0.0986])

# 对 batch 的 logits: shape (B, num_classes)
batch_logits = torch.randn(4, 10)   # 4 个样本，10 个类别
batch_probs = F.softmax(batch_logits, dim=-1)  # 每行独立做 softmax
mmit 关联：MoLE 路由器使用 softmax 选择专家（softmax 将路由分数转为权重）。Attention 层中 softmax 将 Q·K^T 转为注意力权重。

1.3 Cross-Entropy Loss — 训练的目标

是什么：CE = -log(P(正确类别))。衡量模型对正确答案的预测质量。模型越"确定"正确答案，loss 越低。

直觉：如果模型给正确 token 90% 概率 → loss = -log(0.9) = 0.105（很好！）。如果只给 10% → loss = -log(0.1) = 2.302（很差！）。

预测概率 P(正确)Loss = -log(P)含义
0.014.605几乎完全错误
0.102.302很差
0.500.693一般（随机猜二选一）
0.900.105很好
0.990.010非常好
VLM 中的 Cross-Entropy：token 级别的预测
token: <img> [vis1] [vis2] 描述 这张 图片 中有 一只 猫
label: -100 -100 -100 -100 -100 图片 中有 一只 猫
绿色虚线框 = label 为 -100，不计算 loss（指令/图像 token）；黄色/红色 = 计算 loss 的回答 token
PyTorch 用法
# logits: (B, seq_len, vocab_size)，targets: (B, seq_len)
logits = model(input_ids)                        # 模型输出
loss = F.cross_entropy(
    logits.view(-1, vocab_size),                # 展平为 (B*seq_len, V)
    targets.view(-1),                             # 展平为 (B*seq_len,)
    ignore_index=-100                             # -100 位置不计 loss
)
# 内部自动做 softmax + 取 log + 取负号，不需要手动 softmax！
mmit 关联：标准 SFT 中，指令 token 的 label 设为 -100（忽略）。L2T（Learn to Tokenize）改变了这一做法 —— 让视觉 token 也参与 loss 计算。Lavender loss 是 mmit 中一种改进的 loss 计算方式。

1.4 Dropout — 随机丢弃防过拟合

是什么：训练时随机将 p% 的值归零，剩余值乘以 1/(1-p) 进行缩放。推理时什么都不做。

为什么：强制冗余 —— 模型不能依赖任何单一神经元，必须学到更鲁棒的特征。

Dropout 示例（p=0.4）
训练时输入: 0.5 0.3 0.8 0.2 0.6
随机 mask: ✓ ✗ ✓ ✗ ✓
输出 (÷0.6): 0.83 0.0 1.33 0.0 1.0
缩放因子 1/(1-p) = 1/0.6 ≈ 1.667，保证训练和推理时期望值一致
PyTorch 用法
dropout = nn.Dropout(p=0.05)          # 5% 的概率丢弃
identity = nn.Identity()              # 什么都不做的模块（dropout=0 时替代）

# 训练时
model.train()
out = dropout(x)                       # 随机丢弃

# 推理时
model.eval()
out = dropout(x)                       # 等价于 identity，不丢弃
mmit 关联：MoReS 和 ReFT 使用 nn.Dropout(0.05) 在 delta 投影回原空间之前添加正则化。如果 dropout=0，则用 nn.Identity() 替代，避免不必要的计算。

1.5 Linear Layer — 神经网络的基本单元

是什么：y = x @ WT + b。矩阵乘法 + 偏置。这是神经网络中最基本、最重要的操作。

形状规则：输入 (B, in_features)，权重 W (out_features, in_features)，输出 (B, out_features)。

nn.Linear 矩阵乘法
x: [1, 4096] × WT: [4096, 1] + b: [1] = y: [1, 1]
MoReS 的 steering 层: nn.Linear(4096, 1)，将 4096 维向量压缩为 1 个标量
nn.Linear vs F.linear
nn.Linear(in, out)
Module：自带 W 和 b vs F.linear(x, W, b)
函数：你自己提供 W
PyTorch 用法
# nn.Linear — 自动管理权重
layer = nn.Linear(4096, 1, bias=True)
y = layer(x)                           # x: (B, 4096) → y: (B, 1)

# F.linear — 用自定义参数做矩阵乘法
R = nn.Parameter(torch.empty(1, 4096))
y = F.linear(h, R)                    # h: (B, 4096), R: (1, 4096) → y: (B, 1)
API类型权重存储典型用法
nn.Linear(in, out)Module内部管理 weight, bias标准网络层
F.linear(x, W, b)函数需外部提供自定义参数（如 nn.Parameter）做乘法
mmit 关联：MoReS 使用 nn.Linear(4096, 1) 做 steering 标量输出，同时用 F.linear(h, R) 将隐藏状态投影到 R 定义的子空间。Linear 是所有 PEFT 方法的核心构件。

1.6 Embedding — 把整数变成向量

是什么：一个查找表。Token ID = 42 → 取权重矩阵的第 42 行。没有计算，就是查表。

形状：词表大小 V，嵌入维度 D → 权重矩阵 (V, D)。输入 token IDs (B, seq_len) → 输出 (B, seq_len, D)。

Embedding 查找过程
词表 (5×3): 0: [0.2, 0.5, 0.1] 1: [0.8, 0.3, 0.7] 2: [0.1, 0.9, 0.4] 3: [0.6, 0.2, 0.8] 4: [0.3, 0.7, 0.5]
输入 IDs: 2 0 4
输出向量: [0.1, 0.9, 0.4] [0.2, 0.5, 0.1] [0.3, 0.7, 0.5]
PyTorch 用法
emb = nn.Embedding(num_embeddings=32000, embedding_dim=4096)
# emb.weight.shape = (32000, 4096)  — 32K 词表，每个词 4096 维

token_ids = torch.tensor([[5, 12, 8]])   # (1, 3)
vectors = emb(token_ids)                     # (1, 3, 4096)
mmit 关联：文本 token 通过 Embedding 层变成向量。视觉 token 来自 CLIP 编码器 + 投影层（MLP），然后与文本 embedding 拼接（concatenate），形成统一的 (B, seq_len, D) 输入送入 LLaMA。

Day 2: 训练层

2.1 Weight Initialization — 初始化权重

为什么重要：坏的初始化 → 梯度爆炸或消失 → 训练失败。好的初始化让信号在前向和反向传播中保持稳定的幅度。

方法公式/策略适用场景mmit 中的使用
Kaiming (He)方差 = 2/fan_inReLU 网络的标准选择通用默认初始化
正交初始化行向量两两正交，保持向量长度需要保持信息不失真MoReS 的 R 矩阵
零初始化全部为 0"从零干预开始"MoReS 的 W 矩阵（初始无干预）
单位矩阵初始化对角线为 1"从恒等变换开始"ReFT 的 W（初始为恒等映射）
初始化对梯度的影响（通过 10 层网络）
坏初始化: 层1: 1.0 → 层5: 0.001 → 层10: 0.000001 ← 梯度消失！
好初始化: 层1: 1.0 → 层5: 0.95 → 层10: 0.90 ← 梯度稳定 ✓
PyTorch 用法
R = nn.Parameter(torch.empty(1, 4096))
W = nn.Parameter(torch.empty(4096, 4096))

nn.init.orthogonal_(R)                  # 正交初始化（MoReS 的 R）
nn.init.zeros_(W)                       # 零初始化（MoReS 的 W）
nn.init.eye_(W)                         # 单位矩阵（ReFT 的 W）
nn.init.kaiming_uniform_(W)             # Kaiming 初始化

# 带下划线 _ 表示原地操作（in-place），直接修改张量
mmit 关联：MoReS 的设计哲学："从零开始干预"。R 正交初始化保证投影不失真，W 零初始化保证初始 delta=0（不改变原始模型行为）。ReFT 用单位矩阵初始化 W，表示"初始时做恒等变换"。

2.2 nn.Parameter — 可训练参数

是什么：将一个普通张量包装成"可训练参数"，优化器能自动发现并更新它。

vs 普通张量：普通 tensor 不会出现在 model.parameters() 中，优化器看不到它，也不会更新。

nn.Parameter vs 普通 Tensor 的区别
self.R = nn.Parameter(tensor)
✓ 出现在 model.parameters()
✓ 优化器会更新
✓ 保存到 state_dict vs self.R = tensor
✗ 不在 model.parameters()
✗ 优化器看不到
✗ 不保存到 state_dict
PyTorch 用法
class MoReS(nn.Module):
    def __init__(self, hidden_size=4096):
        super().__init__()
        # nn.Parameter — 优化器能找到并更新
        self.R = nn.Parameter(torch.empty(1, hidden_size))
        nn.init.orthogonal_(self.R)

        # nn.Linear 内部也是用 nn.Parameter 存储 weight 和 bias
        self.steering = nn.Linear(hidden_size, 1)

# 验证
m = MoReS()
for name, param in m.named_parameters():
    print(name, param.shape)
# R                    torch.Size([1, 4096])
# steering.weight      torch.Size([1, 4096])
# steering.bias        torch.Size([1])
mmit 关联：MoReS 的 R 矩阵不是放在 nn.Linear 里的，而是直接作为 nn.Parameter 创建。这是当你需要自定义可学习变量（不是标准层的权重）时的做法。

2.3 requires_grad — 控制谁被训练

是什么：每个参数上的布尔标志。True = 计算梯度并更新。False = 冻结，不计算梯度。

核心思想：PEFT 方法的本质就是 —— 冻结大部分参数（requires_grad=False），只训练少量参数（requires_grad=True）。

不同训练策略的 requires_grad 设置
Full FT: Embed ✓ Attn ✓ FFN ✓ Head ✓ ← 全部训练（最贵）
Freeze: Embed ✗ Attn ✗ FFN ✗ Head ✓ ← 只训练最后一层
LoRA: Embed ✗ Attn ✗ A,B ✓ Head ✗ ← 只训练插入的小矩阵
MoReS: Embed ✗ Attn ✗ R,W ✓ Head ✗ ← 只训练 R 和 W
PyTorch 用法
# 冻结整个模型
for param in model.parameters():
    param.requires_grad = False

# 解冻特定部分
for param in model.adapter.parameters():
    param.requires_grad = True

# 查看可训练参数数量
trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
total = sum(p.numel() for p in model.parameters())
print(f"可训练: {trainable}/{total} ({trainable/total*100:.2f}%)")
mmit 关联：所有 PEFT 方法都遵循同一模式：先冻结全部基础模型参数，再将 adapter 参数设为 requires_grad=True。stage_runner.py 中统一管理这个过程。

2.4 Adam / AdamW — 优化器

是什么：SGD 的进阶版。为每个参数维护独立的动量和自适应学习率。

AdamW vs Adam：AdamW 将权重衰减（weight decay）从梯度更新中解耦，效果更好。现代训练几乎都用 AdamW。

Adam 更新步骤
梯度 g → 一阶动量 m = β₁m + (1-β₁)g
梯度的移动平均（方向）
梯度 g → 二阶动量 v = β₂v + (1-β₂)g²
梯度平方的移动平均（大小）
参数更新: θ -= lr × m̂ / √(v̂ + ε)
m̂, v̂ 是偏差修正后的值
超参数默认值含义
lr1e-4学习率（最重要的超参数）
betas(0.9, 0.999)一阶/二阶动量的衰减系数
weight_decay0.01权重衰减（L2 正则化）
eps1e-8防止除零的小常数
SGD vs Adam 在 2D 损失面上的轨迹 ┌──────────────────────────┐ │ ╭──╮ │ │ SGD │ │ ←目标最小值 │ │ ╱╲╱╲╱╲╯ │ │ │ ╱ │ │ │ ╱ Adam │ │ │╱ ─────→──╯ │ │起点 │ └──────────────────────────┘ SGD: 震荡前进 Adam: 平滑收敛
PyTorch 用法
# 基本用法
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=1e-4,
    weight_decay=0.01
)

# 参数组（不同部分不同学习率）
optimizer = torch.optim.AdamW([
    {"params": base_params, "lr": 1e-5},    # 基础模型小学习率
    {"params": adapter_params, "lr": 1e-3},  # adapter 大学习率
])
mmit 关联：stage_runner.py 使用 AdamW，并为不同部分设置不同学习率（参数组）。adapter 通常用比基础模型大 10-100 倍的学习率。

2.5 Gradient Clipping — 防止梯度爆炸

是什么：如果所有参数梯度的总范数超过阈值，按比例缩小所有梯度。

公式：如果 ||g|| > max_norm，则 g = g × max_norm / ||g||

为什么：偶尔出现的异常大梯度会导致参数剧烈变化，破坏已学到的东西。裁剪保持训练稳定。

梯度裁剪示意（max_norm=1.0）
裁剪前: 梯度向量 ||g||=3.0
超出范围！ → 裁剪后: g × (1.0/3.0)
||g||=1.0，方向不变
裁剪前: 梯度向量 ||g||=0.5
未超出，不变 → 裁剪后: 保持不变
||g||=0.5
PyTorch 用法
# 训练循环中，在 optimizer.step() 之前调用
loss.backward()

# 裁剪梯度范数到 max_norm
torch.nn.utils.clip_grad_norm_(
    model.parameters(),
    max_norm=1.0
)

optimizer.step()
optimizer.zero_grad()
mmit 关联：stage_runner.py 在每次 optimizer.step() 之前调用 clip_grad_norm_，通常 max_norm=1.0。这是大模型训练的标配操作。

2.6 Gradient Accumulation — 小显存模拟大 batch

是什么：运行 N 次前向+反向传播，累积梯度，然后做一次优化器更新。

效果：等效 batch size = micro_batch_size × accumulation_steps。

为什么：显存不够放 batch_size=32？用 batch_size=4 × 累积 8 步 = 等效 32。

梯度累积过程（accumulation_steps=4）
micro-batch 1
forward + backward → micro-batch 2
forward + backward → micro-batch 3
forward + backward → micro-batch 4
forward + backward → optimizer.step()
更新参数！
4 个 micro-batch 的梯度累加后，做一次参数更新。等效 batch = 4 × micro_batch_size
PyTorch 用法
accumulation_steps = 8
optimizer.zero_grad()

for i, batch in enumerate(dataloader):
    logits = model(batch["input_ids"])
    loss = F.cross_entropy(logits, batch["labels"])
    loss = loss / accumulation_steps         # ← 关键！除以累积步数
    loss.backward()                          # 梯度累加到 .grad

    if (i + 1) % accumulation_steps == 0:
        torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
        optimizer.step()                    # 更新参数
        optimizer.zero_grad()               # 清零梯度，准备下一轮
注意：loss 必须除以 accumulation_steps！否则梯度会是正确值的 N 倍，等价于学习率放大了 N 倍。这是初学者最常犯的错误之一。

2.7 Learning Rate Schedule — 学习率调度

Cosine with warmup：先线性地从 0 升到目标 lr（warmup），然后按余弦曲线从 lr 降到 0。

为什么要 warmup：训练初始，权重是随机的，如果一开始就用大学习率，会产生巨大的不稳定更新。先"热身"让模型稳定下来。

学习率调度：Cosine with Warmup lr 1e-4 | ╭──╮ | ╱ ╲ | ╱ ╲ | ╱ ╲ 5e-5 | ╱ ╲ |╱ ╲ ╱ ╲ 0 ╱────────────────╲──→ step 0 warmup total_steps │←─ 线性 ─→│←── 余弦衰减 ──→│
学习率随训练进度变化
Step 0
lr=0 → Step 50
lr=5e-5 → Step 100
lr=1e-4 (峰值) → Step 500
lr=7e-5 → Step 1000
lr≈0
warmup_steps=100, total_steps=1000
PyTorch 用法
import math

def cosine_warmup_lambda(step, warmup_steps, total_steps):
    """返回学习率乘数 [0, 1]"""
    if step < warmup_steps:
        return step / warmup_steps              # 线性 warmup
    progress = (step - warmup_steps) / (total_steps - warmup_steps)
    return 0.5 * (1 + math.cos(math.pi * progress))  # 余弦衰减

optimizer = torch.optim.AdamW(params, lr=1e-4)
scheduler = torch.optim.lr_scheduler.LambdaLR(
    optimizer,
    lr_lambda=lambda s: cosine_warmup_lambda(s, 100, 1000)
)

# 每个 optimizer.step() 后调用
optimizer.step()
scheduler.step()       # 更新学习率
mmit 关联：stage_runner.py 实现了 cosine warmup schedule。典型配置：warmup 占总步数的 3-10%。学习率调度对训练效果影响很大 —— 不用 warmup 容易在开始阶段就"炸掉"。
