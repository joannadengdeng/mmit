从论文 Idea 到代码实现

一张地图 · 六个阶段 · 五个插槽 — 你只需要找到 idea 插在哪里

你遇到的真正问题

不是"看不懂论文"，而是缺一张地图。 你能看懂 LiLoRA 的公式 ΔW = (α·B₀ + (1-α)·B̃·Ã)·A，但不知道这个公式放在整个训练系统的哪个位置，所以不知道该写哪个文件、哪个函数、和哪些已有代码对接。

解决方法：先理解完整的训练流水线有哪些阶段，然后对任何一篇论文，问一个问题：
"这篇论文改的是流水线的哪个阶段？"

关键认知：论文的 idea 永远只改其中 1-2 个阶段，其他阶段不变。 你不需要从零写一个训练系统。你只需要找到论文改的是哪个阶段，然后只改那个阶段的代码。

五个"插槽"：论文 idea 只会插在这些位置

插槽 A: 模型架构改造
"模型的哪些参数/模块被修改了？"
  • LoRA: 在 Linear 旁边加 A·B 低秩矩阵
  • LiLoRA: 把 B 拆成共享 B₀ + 任务 B̃·Ã
  • MoReS: 在层输出后加正交干预模块
  • Freeze: 不加模块，只选择解冻哪些层
对应代码: prepare_model() → 阶段 2
插槽 B: 损失函数
"loss 怎么算的？和标准交叉熵有什么不同？"
  • 标准: L = CrossEntropy(logits, labels)
  • MoReS: L = CE + λ·正交罚项
  • L2T: 标准 CE，但改了哪些 token 参与计算
  • 对比学习: L = CE + InfoNCE
对应代码: compute_loss() → 阶段 4
插槽 C: 标签/数据处理
"数据进模型之前，做了什么特殊预处理？"
  • 标准: 只对 response token 算 loss（instruction 置 -100）
  • L2T: unmask 非模板的 instruction token
  • 数据增强: 随机裁剪、混合
  • 课程学习: 按难度排序样本
对应代码: preprocess_labels() / collate → 阶段 3-4
插槽 D: 优化策略
"optimizer/scheduler 有什么特别？"
  • 标准: AdamW + cosine schedule
  • 不同参数组不同 LR
  • 梯度裁剪策略
  • EMA（指数移动平均）
对应代码: optimizer 配置 → 阶段 4
插槽 E: 保存/加载格式 — "checkpoint 里存了什么？加载时怎么恢复？"
LoRA 存适配器权重 | LiLoRA 存每层的 LiLoRA state | MoReS 存干预模块 + projector | Full FT 存整个模型
对应代码: save_checkpoint() + load_for_inference() → 阶段 5-6

实战：用这个框架分析我们接触过的每篇论文

论文 一句话 idea A 架构 B 损失 C 数据 D 优化 E 存储 关键代码位置
LoRA 低秩 A·B 矩阵旁路 ✔ 主要 - - - ✔ prepare_model() 注入 PeftModel
LiLoRA 共享 A + 分解 B + 正则 ✔ 主要 - - - ✔ LiLoRALinear + hook 注入
L2T 也对 instruction 算 loss - - ✔ 主要 - - preprocess_labels() 改 mask
MoReS 正交低秩干预 + 投影器 ✔ ✔ - - ✔ OrthogonalIntervention + 正交罚项
DoRA 分解权重为幅度+方向 ✔ 主要 - - - ✔ PEFT 库内部实现
QLoRA 4-bit 量化 + LoRA ✔ - - ✔ NF4 ✔ BitsAndBytesConfig + LoRA
发现了吗？ 大多数论文只动了一个插槽。L2T 只改了标签处理（C），LoRA/LiLoRA 只改了模型架构（A），MoReS 改了两个（A+B）。没有一篇论文需要你从零重写整个训练系统。

读论文时的 5 个问题清单

拿到任何一篇 VLM/PEFT 论文，按顺序问自己这 5 个问题：

#问题对应插槽怎么在论文中找答案
1模型被怎样修改了？
加了什么模块？冻结了哪些？		A 架构看 "Method" 部分的架构图、公式 ΔW = ...。如果有新模块，就是一个 nn.Module。冻结的部分用 requires_grad=False。
2Loss 和标准 CE 有什么不同？
有额外的正则项吗？		B 损失看 "Training Objective" 或 "Loss Function"。如果只有 LCE，则不需要改。如果有 L = LCE + λ·Lreg，则需要写 compute_loss()。
3数据/标签做了什么特殊处理？C 数据看 "Data Processing" 或 "Label Masking"。如果论文说"也对 instruction 计算 loss"，就是改 labels tensor。
4优化器/学习率有什么特殊设计？D 优化看 "Implementation Details"。大部分论文用标准 AdamW + cosine，这个插槽经常不需要改。
5要保存什么？怎么加载？E 存储取决于架构：LoRA 系只存适配器，Full FT 存整个模型，自定义模块存自定义 state_dict。

完整的思维路径：从读论文到写代码

【读论文】 读 Abstract + Method section | v 【定位插槽】 问自己: 这篇论文改了流水线的哪个阶段？ | "模型架构有新模块" ──→ 插槽 A ──→ 写 nn.Module + prepare_model() "Loss 有额外项" ──→ 插槽 B ──→ 写 compute_loss() "标签/数据做了处理" ──→ 插槽 C ──→ 写 preprocess_labels() "优化策略不同" ──→ 插槽 D ──→ 改 optimizer/scheduler 配置 "存储格式特殊" ──→ 插槽 E ──→ 写 save_checkpoint() + load_for_inference() | v 【翻译公式】 论文里的每个数学符号，对应一个 PyTorch 操作： 矩阵 W ∈ R^{m×n} → nn.Parameter(torch.zeros(m, n)) 或 nn.Linear(n, m) 矩阵乘 W·x → F.linear(x, W) sigmoid(z) → torch.sigmoid(z) cos 相似度 → F.cosine_similarity(a, b) L2 范数 ||W||² → W.pow(2).sum() 可学习标量 α → nn.Parameter(torch.tensor(init_val)) | v 【注入模型】 把新模块挂到已有模型上： register_forward_hook → 不改原模型，在输出上"加一笔" (LiLoRA, MoReS) PeftModel.from_pretrained → 用 PEFT 库自动注入 (LoRA, QLoRA, DoRA) 直接修改 requires_grad → 选择性解冻 (Freeze Tuning) | v 【其他不变】 训练循环、数据加载、评估 —— 大部分代码复用，只改动了上面定位的插槽

实例演练：假设你第一次读到 LiLoRA

模拟你的思考过程，一步步走：

步骤你的思考找到的答案代码动作
1 读 Abstract："共享 A 矩阵，分解 B 矩阵" 这是在修改模型架构 → 插槽 A 需要写一个新的 nn.Module
2 看公式：ΔW = (α·B₀ + (1-α)·B̃·Ã)·A 5 个可学习参数：A, B₀, α, B̃, Ã 在 __init__ 中声明 5 个 nn.Parameter 或 nn.Linear
3 这个 ΔW 加在哪里？ "每个目标 Linear 层的输出上" 用 register_forward_hook 把 δ 加到 base_output 上
4 Loss 有特殊的吗？ 有余弦正则化 Lreg（用于持续学习） 单任务场景可先用标准 CE；多任务时再加 compute_loss()
5 数据处理有特殊的吗？ 没有，标准 VIT 数据格式 不需要改数据代码
6 要保存什么？ 只需保存 LiLoRA 模块的参数（基座不变） 遍历 self._lilora_modules，每个调 state_dict() 保存
7 加载推理时呢？ 加载基座 → 重建 LiLoRA 结构 → 填入权重 prepare_model() 然后 load_state_dict()
总结：LiLoRA 只改了插槽 A（架构）和 E（存储），其余完全复用。 你需要写的代码 = 一个 LiLoRALinear(nn.Module) 约 35 行 + hook 注入约 15 行 + save/load 约 25 行。训练循环、数据加载、评估全部不用动。

你的行动清单：下次读论文时

5 步法：
1. 画流水线 — 在脑中过一遍 6 个阶段：加载 → 改造 → 数据 → 训练 → 保存 → 推理
2. 定位插槽 — 论文的核心贡献改了哪个阶段？（多数论文只改 1 个）
3. 数公式符号 — 论文引入了几个新的可学习参数？每个的维度是什么？
4. 翻译成 Module — 每个参数 = nn.Parameter，每个矩阵乘法 = F.linear，每个激活 = F.xxx
5. 挂上去 — 用 hook / PEFT / requires_grad 把新模块连接到已有模型。其他阶段照搬。
