从零理解 PEFT 与训练框架

从"什么是微调"开始，一步步到你的 mmit 训练框架

什么是微调（Fine-tuning）？

类比：一个人从小读了很多书（预训练），现在你想让他当一个眼科医生（下游任务）。
你不需要让他重新从幼儿园学起，只需要让他去医学院进修（微调）。

大语言模型（LLM）经过海量文本预训练后，已经"懂"了语言。微调就是在预训练好的模型基础上，用你的特定数据再训练一下，让它学会新技能。

例子

预训练的 LLaMA 模型 → 用指令数据微调 → 变成能回答问题的 Vicuna
预训练的 CLIP + LLaMA → 用图文指令数据微调 → 变成能看图回答的 LLaVA

问题来了：模型有几十亿个参数。全部重新训练一遍，需要多大的代价？

为什么不能全部训练？

7B 参数模型

约 70 亿个参数 · 存储需要 ~14GB (FP16) · 训练时显存需要 ~56GB（含梯度+优化器状态）· 需要 4 张 A100 (40GB)

类比：为了教一个大学教授做眼科手术，你不需要改变他所有的知识（语文、数学、历史……），只需要在他现有知识的基础上，添加一小部分新的手术技能。

全量微调（Full Fine-tuning）的问题：

  • 显存爆炸 — 每个参数需要存储：权重 + 梯度 + 优化器状态 ≈ 4x 存储
  • 灾难性遗忘 — 全部训练容易忘记预训练学到的知识
  • 存储浪费 — 每个任务保存一份完整模型副本

解决方案：冻结大部分参数，只训练一小部分 → 这就是 PEFT。

Adapter（适配器）到底是什么？

类比：想象一个插座转换器。你不需要改变整个电力系统（预训练模型），只需要在插座上加一个小小的转换器（adapter），就能兼容新的设备（新任务）。

Adapter（适配器）是一个统称，指的是插入到预训练模型中的一小块可训练模块。

核心思想

1. 冻结原始模型的所有参数（不动它们）
2. 在模型内部插入一些新的小模块（adapter）
3. 只训练这些新插入的小模块
4. 保存时只需要保存这些小模块（通常只有原模型的 0.1%~1%）

不同的 PEFT 方法，本质上就是：在模型的不同位置，用不同方式插入 adapter。

方面全量微调Adapter（PEFT）
训练参数量100%0.1% ~ 3%
显存需求极高低很多
保存大小整个模型 (14GB+)几十 MB
多任务每个任务一份完整模型每个任务只存一份 adapter
遗忘风险高低（原始权重不变）

三种主流 Adapter 方法

所有 PEFT 方法都在做同一件事：在冻结的模型中插入可训练模块。区别在于插在哪里、怎么插。

Adapter Layer — 串联插入

不需要全部记住！实际中用得最多的是 LoRA，因为它不增加推理延迟、参数量少、效果好。下一节详细讲 LoRA。

LoRA — 最流行的 PEFT 方法

类比：一本教科书（原始权重矩阵）已经印好了，你不能改它。但你可以在旁边贴一张小便签纸（LoRA 矩阵），每次读的时候把便签上的内容也加进去。便签很小但很有效。

关键参数

参数含义典型值
rank (r)低秩矩阵的秩，越大表达能力越强但参数越多8 ~ 64
alpha (α)缩放因子，控制 LoRA 输出的权重（通常 α/r）16 ~ 64
target_modules在哪些层注入 LoRA（如 q_proj, v_proj）注意力层的 Q 和 V
dropoutLoRA 层的 dropout 比率0.05

为什么 LoRA 这么流行？

1. 推理零开销 — 训练完可以把 LoRA 矩阵 merge 回原权重，推理时没有额外计算
2. 参数极少 — 7B 模型加 LoRA (r=16) 只需训练 ~4M 参数（0.06%）
3. 效果好 — 在很多任务上接近全量微调的效果
4. 可叠加 — 同一个基座模型可以加载不同的 LoRA adapter 做不同任务

mmit 训练框架 — 把一切串起来

现在你理解了 adapter 的概念，来看 mmit 框架是怎么让研究员随意切换不同的 adapter 方法的：

核心思路：Trainer 不管具体策略

类比：Trainer 就像一个健身房教练。他负责安排你的训练计划（几组、几次、休息多久），但他不管你用什么器械（LoRA？Adapter？RepAdapter？）。器械的选择由 PEFTStrategy 决定。

三个角色分工

角色类做什么谁实现
教练Trainer管理训练循环、optimizer、checkpoint框架（已写好）
器械PEFTStrategy决定在模型哪里加什么 adapter研究员
评分标准LossFunction计算 loss（模型表现多好）研究员

使用方式

# 1. 选择策略（通过配置文件的 name 字段）
config = TrainingConfig(
    peft = PEFTConfig(name="lora", rank=16),  # ← 换个名字就换策略
    loss = "causal_lm",
)

# 2. Trainer 通过 Registry 自动找到对应的策略类
trainer = Trainer.from_config(method, config)
#   → Registry 查找 "lora" → 实例化 LoRAStrategy
#   → Registry 查找 "causal_lm" → 实例化 CausalLMLoss

# 3. 开始训练（Trainer 按顺序调用策略的方法）
trainer.train(dataset)
#   内部：
#   peft.wrap_model(method, config)   → 策略注入 adapter
#   peft.trainable_params(method)     → 策略返回要训练的参数
#   for batch: forward → loss → backward → step
#   peft.save_adapter(method, path)   → 策略保存 adapter

换一个策略有多简单？

# 想用 LoRA？
config.peft.name = "lora"

# 想用 LoRA-in-LoRA？只需要：
# 1. 研究员写好 LoRAInLoRAStrategy 类
# 2. 注册：registry.register("peft", "lora-in-lora", LoRAInLoRAStrategy)
# 3. 切换配置：
config.peft.name = "lora-in-lora"

# Trainer 的代码？一行都不用改。
