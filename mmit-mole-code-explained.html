Block 1: LoRAExpert — 一个专家长什么样 (Line 37-50)

class LoRAExpert(nn.Module):
    def __init__(self, in_features, out_features, rank, dropout=0.0):
        self.lora_A = nn.Linear(in_features, rank, bias=False)    # 降维: 4096 → 8
        self.lora_B = nn.Linear(rank, out_features, bias=False)   # 升维: 8 → 11008
        nn.init.kaiming_uniform_(self.lora_A.weight, a=5**0.5)
        nn.init.zeros_(self.lora_B.weight)                          # B=0 → 初始 delta=0

    def forward(self, x):
        return self.lora_B(self.lora_A(self.dropout(x)))             # x→A→B = delta

Block 2: MoLELayer — Router + 多专家 dispatch (Line 53-122)

2.1 __init__: 冻结原 Linear + 创建 4 expert + router

class MoLELayer(nn.Module):
    def __init__(self, base_linear, num_experts, rank, alpha, dropout):
        self.base_linear = base_linear              # 原始 nn.Linear, 冻结
        self.scaling = alpha / rank                  # 16/8 = 2.0
        for p in base_linear.parameters():
            p.requires_grad = False                  # 冻结原始权重

        self.experts = nn.ModuleList([
            LoRAExpert(in_f, out_f, rank, dropout) for _ in range(num_experts)])  # 4 个
        self.router = nn.Linear(in_f, num_experts, bias=False)  # (4096 → 4)

2.2 forward: router 选专家 + dispatch

def forward(self, x):                         # x: (batch, seq, 4096)
    base_out = self.base_linear(x)              # frozen 输出: (B, seq, 11008)

    router_logits = self.router(x.detach())     # (B, seq, 4) ← detach!
    routing_probs = F.softmax(router_logits, dim=-1)  # 归一化成概率
    expert_idx = routing_probs.argmax(dim=-1)   # (B, seq) top-1 选择

    # dispatch: 每个 expert 处理分配给它的 token
    x_flat = x.reshape(-1, x.shape[-1])        # (620, 4096)
    idx_flat = expert_idx.reshape(-1)            # (620,)
    delta = torch.zeros(...)                     # (620, 11008)

    for i, expert in enumerate(self.experts):
        mask = (idx_flat == i)                   # 哪些 token 给 expert i
        if mask.any():
            delta[mask] = expert(x_flat[mask])   # 只跑选中的 token

    return base_out + delta.reshape(...) * self.scaling  # frozen + expert delta × 2.0

API做什么为什么
x.detach()切断梯度不让 loss 的梯度通过 router 回传到 x，稳定训练
F.softmax(_, dim=-1)概率归一化[2.1, 0.5, -0.3, 0.8] → [0.55, 0.11, 0.05, 0.15, ...]
.argmax(dim=-1)选最大的索引概率 [0.12, 0.65, 0.15, 0.08] → 选 expert 1
F.one_hot(idx, N)整数→one-hotexpert_idx=1 → [0,1,0,0]，用于 balance loss
x.detach() 为什么重要：如果不 detach, loss 的梯度会通过 router → x → 回传到整个模型, 让 frozen 参数也收到梯度（违反冻结设计）。detach 让 router 只根据 x 的"值"做决定, 但不把梯度传回 x。

为什么 MLP 用 MoE 而 Attention 不用？论文的 insight: 数据冲突主要在 MLP 层（负责"记忆"知识），Attention 层主要做"关注"不太受数据类型影响。所以 MLP 用多专家分流, Attention 用单 LoRA 就够。

Block 6: _load_balance_loss — 均匀使用专家 (Line 279-303)

def _load_balance_loss(self):
    for layer in self._mole_layers:              # 96 个 MoLELayer
        probs = layer._routing_probs.reshape(-1, 4)  # (620, 4) softmax 概率
        mask = layer._expert_mask.reshape(-1, 4)    # (620, 4) one-hot 选择
        f = mask.mean(dim=0)                       # (4,) 每个 expert 分到的比例
        P = probs.mean(dim=0)                      # (4,) 每个 expert 的平均概率
        total_loss += (f * P).sum() * num_experts   # N × Σ(fᵢ × Pᵢ)
    return total_loss / count
