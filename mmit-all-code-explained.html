Full Fine-Tuning (全参数微调)

full_ft.py 图解 — FullFTMethod (Line 1-80)

所有 PEFT 方法中最简单的一个，仅 ~80 行代码。全参数微调：直接训练模型所有权重。基于 LLaVA-1.5-7B (约 70 亿参数)。

Block 1: _prepare_model_impl — 设置全部参数可训练 (Line 42-48)

def _prepare_model_impl(self, model, processor, config):
    for p in model.parameters():
        p.requires_grad = True                       # 每个参数都设为可训练

    trainable = sum(p.numel() for p in model.parameters())
    info = f"Full fine-tuning: all {trainable:,} parameters trainable"
    return model, info

这里用到的 PyTorch API

API做什么为什么用它
model.parameters()返回模型所有参数的迭代器遍历每一个 nn.Parameter，逐个设置 requires_grad
p.requires_grad = True标记该参数需要计算梯度全参数微调 = 所有参数都参与反向传播和优化器更新
p.numel()返回 tensor 中元素的总数统计可训练参数量，用于日志输出
显存对比：Full FT 训练 7B 模型需要 ~56GB+ 显存（模型 14GB + 梯度 14GB + 优化器状态 28GB）。LoRA 只训练 ~4M 参数，显存仅需 ~16GB。MoReS 只训练 ~8K 参数，显存更低。Full FT 推荐使用 >=24GB GPU 或分布式训练。
freeze_patterns：可以通过配置 freeze_patterns: ["vision_tower"] 选择性冻结某些模块（如视觉编码器），在基类 _prepare_model 中调用 _freeze_by_patterns 实现。这里的 _prepare_model_impl 先把所有参数设为可训练，然后基类再根据 freeze_patterns 冻结匹配的模块。

_ce_loss = CrossEntropyLoss()

def compute_loss(self, model, batch, outputs):
    return _ce_loss.compute(model, batch, outputs)   # 标准交叉熵

def get_trainable_params(self, model):
    return [{ "params": [p for p in model.parameters() if p.requires_grad] }]

Block 1: _find_transformer_layers — 查找 decoder 层 (Line 23-33)

def _find_transformer_layers(model: nn.Module) -> list[nn.Module]:
    """Find the sequential list of transformer layers in a model."""
    for attr in ("model.layers", "transformer.h", "gpt_neox.layers"):
        obj = model
        try:
            for part in attr.split("."):   # e.g. "model" → "layers"
                obj = getattr(obj, part)
            return list(obj)              # nn.ModuleList → list
        except AttributeError:
            continue
    return []

def save_checkpoint(self, model, processor, path, metadata):
    os.makedirs(path, exist_ok=True)
    # Step 1: 找出所有可训练参数的名字
    trained_names = {n for n, p in model.named_parameters() if p.requires_grad}
    # Step 2: 从 state_dict 中只取这些参数
    trainable_state = {k: v for k, v in model.state_dict().items() if k in trained_names}
    # Step 3: 保存 (~880M params → ~1.7GB 文件)
    torch.save(trainable_state, os.path.join(path, "freeze_tuned.pt"))
    processor.save_pretrained(path)
    # Step 4: 保存元数据
    metadata["ft_method"] = self.name
    metadata["trained_param_names"] = sorted(trained_names)
    with open(os.path.join(path, "mmit_meta.json"), "w") as f:
        json.dump(metadata, f, indent=2, ensure_ascii=False)

def load_for_inference(self, path, base_model_id, **kwargs):
    from transformers import AutoProcessor
    try:
        from transformers import AutoModelForImageTextToText as AutoVLM
    except ImportError:
        from transformers import AutoModelForVision2Seq as AutoVLM

    # Step 1: 加载原始 base model (7B 完整权重)
    processor = AutoProcessor.from_pretrained(base_model_id, trust_remote_code=True)
    model = AutoVLM.from_pretrained(
        base_model_id, torch_dtype=torch.bfloat16,
        device_map="auto", trust_remote_code=True,
    )
    # Step 2: 加载微调过的参数, 覆盖 base model 中对应的权重
    state = torch.load(
        os.path.join(path, "freeze_tuned.pt"),
        map_location="cpu", weights_only=True,
    )
    model.load_state_dict(state, strict=False)   # strict=False: 只覆盖有的 key
    model.eval()

def forward(self, x):                         # x: (batch, seq, 4096)
    base_out = self.base_linear(x)              # frozen 输出: (B, seq, 11008)

    router_logits = self.router(x.detach())     # (B, seq, 4) ← detach!
    routing_probs = F.softmax(router_logits, dim=-1)  # 归一化成概率

API做什么为什么用它
nn.Parameter(tensor)把 tensor 注册为可训练参数这样 optimizer 能找到它并更新它。普通 tensor 不会被训练
torch.empty(1, 4096)创建未初始化的 tensor下一行会用正交初始化覆盖，所以初始值无所谓

nn.init.zeros_(tensor)全部填 0W=0 → 训练初期干预效果=0，模型行为不变，慢慢学
nn.Dropout(0.05)训练时随机 5% 值变 0防过拟合。推理时自动关闭
nn.Identity()什么都不做 (y = x)dropout=0 时的占位符，避免 if/else
关键设计：W 初始化为零 → 训练第一步时 Wh = 0，干预 delta ≈ 0，模型输出和原模型一样。然后 W 逐步学出非零值，模型被慢慢引导。这比一开始就大幅修改更安全。

Block 1: 核心概念 — "嵌套"思路 (LoRA-in-LoRA)

论文: Che et al., 2025. 核心问题: 模型依次学 Task 1、Task 2、Task 3 时，学新任务会覆盖旧任务的知识 (灾难性遗忘)。

每个 Stage: 先把上一轮的 LoRA merge 进 base weights (绿虚线 → 蓝框), 再装一个新的小 LoRA (红框). 旧知识不会被覆盖因为已经 "固化" 到权重里了.
为什么能防遗忘? 标准做法 (直接换 LoRA) 会完全丢弃 Task 1 学到的低秩更新. LoRA-in-LoRA 把旧 LoRA merge 进 base weights, 旧知识变成了模型 "本能" 的一部分, 和预训练知识同等地位. 新 LoRA 只在此基础上做微小调整, 不会破坏已有能力.
为什么 inner rank 更小? 每一阶段的增量任务通常比第一个任务简单 (因为基础能力已经有了). r=4 (vs r=8) 参数量减半, 够用且不容易过拟合.

2. 包装器模式 — _get_base + 委托 (Line 42-49, 78-81)

3. build_template_list — 扫描数据集提取模板句 (Line 83-95)

训练前扫描整个数据集, 用 Counter 统计所有 user 轮次中的句子频率. 出现次数最多的 top-K 句子被判定为 模板句 (样板指令), 保持 masked. 其余低频句子被视为 内容性指令, 将被 unmask.

def build_template_list(self, dataset: list, processor, top_k: int = 20):
    sentence_counter: Counter = Counter()
    for sample in dataset:
        if hasattr(sample, "turns"):
            for turn in sample.turns:
                if turn.role in ("human", "user"):
                    for sent in turn.content.replace("\n", ". ").split(". "):
                        sent = sent.strip()
                        if sent:
                            sentence_counter[sent] += 1
    self._templates = {sent for sent, _ in sentence_counter.most_common(top_k)}

4. preprocess_labels — 核心创新, 仅 5 行 (Line 97-101)

这就是 L2T 的 全部算法. 找到 labels 中所有 -100 (被 masked) 的位置, 将它们恢复为原始 input_ids. 效果: 原本不参与 loss 的指令 token 现在也参与了.

def preprocess_labels(self, input_ids, labels, batch_meta=None):
    mask = labels == IGNORE_INDEX       # Step 1: 找到所有 -100 位置
    if mask.any():
        labels = labels.clone()         # Step 2: 克隆避免原地修改
        labels[mask] = input_ids[mask]  # Step 3: 用原始 token id 覆盖 -100
    return labels

def load_for_inference(self, path, base_model_id, **kwargs):
    import json
    meta_path = os.path.join(path, "mmit_meta.json")
    base_name = "lora"                               # 默认 fallback
    if os.path.exists(meta_path):
        with open(meta_path) as f:
            meta = json.load(f)
        base_name = meta.get("l2t_base_method", "lora")
    from mmit.registry import registry
    base_cls = registry.get("training_method", base_name)
    base = base_cls()
    return base.load_for_inference(path, base_model_id, **kwargs)

6. 总结

维度L2T其他方法 (LoRA, MoRes, etc.)
模型架构改动无新增参数 / 修改层
Loss 函数改动无部分方法有自定义 loss
改动对象仅 labels tensor权重 / 梯度 / 前向传播
代码量核心 5 行几十到几百行
可组合性可与任何 base method 叠加通常互斥
推理开销零 (与 base 完全相同)视方法而定
L2T 的独特之处: 它是所有方法中唯一 不改变模型架构, 不改变 loss 函数 的方法 — 它只改变 loss 计算在 哪些 token 上执行. 极简的设计使它可以无缝叠加在 LoRA、QLoRA 或全量微调之上, 作为一个正交维度的增强.
