拼接后的 hidden state — 实际进入 LLM 的样子

在 forward 内部，1 个 <image> token 被展开成 576 个 visual tokens：

[ BOS A chat ... USER: | VIS₁ VIS₂ VIS₃ ... VIS₅₇₆ | \n What are the colors of the buses ? ASSISTANT: The buses ... ]
最终 hidden state shape
position 0~4: 5 个系统 prompt tokens (蓝色) position 5~580: 576 个 visual tokens (橙色 ← MoReS 关心的!) position 581~619: 39 个文本 tokens (白色) 总长度: 620 hidden state shape: (1, 620, 4096)
关键事实：576 个 visual tokens 在 hidden state 里是连续的一段 (位置 5~580)。 MoReS 的目标就是在这 576 个位置上做修正。

2. Vicuna-7B 的 32 层 decoder — MoReS 在哪 9 层 hook

hidden state (1, 620, 4096) 进入 LLM 后会逐层 forward。MoReS 在 positions = "f4+l5" 这 9 层注册了 forward hook：

Layer 0   LlamaDecoderLayer★ MoReS hook
Layer 1   LlamaDecoderLayer★ MoReS hook
Layer 2   LlamaDecoderLayer★ MoReS hook
Layer 3   LlamaDecoderLayer★ MoReS hook (前 4 层 = "f4")
Layer 4普通 forward
Layer 5普通 forward
Layer 6普通 forward
⋮ Layer 7 ~ 25 (19 层) — 普通 forward, 无 hook ⋮
Layer 26普通 forward
Layer 27   LlamaDecoderLayer★ MoReS hook
Layer 28   LlamaDecoderLayer★ MoReS hook
Layer 29   LlamaDecoderLayer★ MoReS hook
Layer 30   LlamaDecoderLayer★ MoReS hook
Layer 31   LlamaDecoderLayer★ MoReS hook (后 5 层 = "l5")
_parse_positions("f4+l5", num_layers=32) 的代码逻辑
f4 = list(range(0, 4)) = [0, 1, 2, 3] # 前 4 层 l5 = list(range(32-5, 32)) = [27, 28, 29, 30, 31] # 后 5 层 target_indices = sorted(f4 + l5) = [0, 1, 2, 3, 27, 28, 29, 30, 31] # 共 9 层会被 hook, 中间 23 层完全不动

每一个 hooked 层的输出都会经过同一个 intervention 模块（共享权重）。 所以参数总量只有一份 intervention 的大小。

整个 LlamaDecoderLayer 的 ~200M 参数全部 frozen。 MoReS 不修改 weight，只是在 layer 输出后做一次加法 (h' = h + delta)。 后续 layer 拿到的是修改后的 h'，所以模型行为发生变化。

4. OrthogonalIntervention 内部 — 用矩阵图展示每一步

4.1 模块的两个参数

R (正交投影)
[ r₁ r₂ r₃ ... r₄₀₉₆ ]
shape: (1, 4096) | 参数: 4096
W (Linear)
weight: [ w₁ w₂ ... w₄₀₉₆ ] bias: [ b ]

初始化：R 用 nn.init.orthogonal_ 初始化（满足 R·Rᵀ ≈ I）， W.weight 和 W.bias 全 0。 所以训练第一步时 delta = 0，模型行为和原模型完全一样，然后慢慢学。

5. 怎么从 620 个 token 里挑出 6 个？— Top 1% by norm

6. 训练时只有 8,193 个参数有梯度

在我们的例子里，loss 是 cross-entropy on "The buses are white and red." 这 9 个 token 的预测。 backward 时梯度只能流到 requires_grad=True 的参数：

参数清单 (LLaVA-1.5-7B + MoReS)
🔒 vision_tower (CLIP): ~300,000,000 FROZEN (无梯度) 🔒 multi_modal_projector: ~25,000,000 FROZEN 🔒 language_model.embed_tokens: ~131,000,000 FROZEN 🔒 language_model.layers (32×): ├─ 4 × attention proj ~67M × 32 FROZEN ├─ 3 × FFN proj ~134M × 32 FROZEN └─ 2 × RMSNorm ~8K × 32 FROZEN 🔒 lm_head: ~131,000,000 FROZEN ──────────────────────────────────────────────── ★ OrthogonalIntervention (共享): R 4,096 TRAINABLE ★ W.weight 4,096 TRAINABLE ★ W.bias 1 TRAINABLE ★ ─────────── 8,193 ★ 唯一有梯度的参数 ──────────────────────────────────────────────── 总计: 7,063,008,193 params, trainable ratio = 0.000116%

梯度流向

backward() 在做什么
loss = CE(logits[answer_pos], labels[answer_pos]) loss.backward() # PyTorch 自动梯度反传: # lm_head ← logits 的梯度 # layer 31 ← lm_head 的梯度 (frozen, 不更新) # intervention(layer 31 output) ← 这里有 hook! # → R.grad, W.weight.grad, W.bias.grad ★ # layer 31 weight ← (frozen, 不更新) # layer 30 ← ... # intervention(layer 30 output) ← 同一个 intervention 又收一次梯度 # ... 9 个 hooked layer 的梯度会累加到同一个 intervention 上 ... # layers 4~26 (普通 forward) ← 没有 hook, 梯度直接流过去 # layer 0~3 ← 又触发 hook # intervention 收到累计 9 次的梯度 optimizer.step() # 只更新 R 和 W (8,193 个参数)

7. 一句话总结

MoReS = 在 LLaVA 的 32 层 decoder 中,
只在 9 个特定层的输出上,
对最重要的 1% visual token,
用一个共享的 8,193 参数小模块,
做一次 rank-1 的正交修正

在我们的例子里：模型看到公交车图 + 问题 → CLIP 出 576 个 visual tokens → 进入 32 层 LLM → 在 9 个层上, top 1% (~6 个) 的视觉 token 被一个 8K 参数的小模块修正方向 → 最终生成 "The buses are white and red." 的概率被微调。

训练时，这 8,193 个参数从 0 学起，慢慢学到"应该把视觉 token 往哪个方向 nudge" 才能让模型回答得更准。
