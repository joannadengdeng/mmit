LoRA — 低秩适配

类比：教科书不能改，但可以在页边贴便签。每次读书时顺便看便签，就学到了新知识。便签很小，不影响书本身。

原理

在 LLM 的 attention 层的权重矩阵旁边，并联一对低秩矩阵 A (d→r) 和 B (r→d)：

y = W·x + (α/r) · B·A·x

W 冻结不动，只训练 A 和 B。r 通常 = 8~64，所以参数量极少（~0.1%）。

在 mmit 中使用

# 内置 LoRAStrategy — 直接用
from mmit.config.schema import TrainingConfig, PEFTConfig
from mmit.training import Trainer

config = TrainingConfig(
    peft=PEFTConfig(
        name="lora",           # ← 内置策略名
        rank=16,                  # 低秩的 r
        alpha=32,                 # 缩放因子 α
        target_modules=["q_proj", "v_proj"],
    ),
    loss="causal_lm",
    learning_rate=2e-5,
    num_epochs=3,
)

trainer = Trainer.from_config(method, config)
trainer.train(dataset)

框架调度流程

框架方法LoRAStrategy 做了什么
wrap_model(method)在 method.llm 上用 HuggingFace peft 注入 LoRA 层，冻结原始权重
trainable_params(method)返回 LoRA 的 A/B 矩阵参数（和可选的 projector 参数）
save_adapter(method)用 peft 库的 save_pretrained 只保存 LoRA 权重
load_adapter(method)用 PeftModel.from_pretrained 加载

Projector 预训练（LLaVA Stage 1）

类比：你请了一个翻译（Projector）。你不需要改变说话的人（Vision）和听的人（LLM），只需要训练翻译学会把"图片语言"翻译成"文字语言"。

原理

LLaVA Stage 1 的目标：让 Projector 学会对齐视觉特征和语言特征空间。

Vision Encoder（CLIP）已经能提取图像特征，LLM 已经能理解语言。
但它们的特征维度不同（CLIP: 1024d, LLM: 4096d）。
Projector 就是一个"翻译器"，把视觉特征投影到 LLM 的空间里。

这个阶段：只训练 Projector，Vision 和 LLM 全部冻结。

在 mmit 中使用

# 自定义策略：只训 Projector
class ProjectorOnlyStrategy(PEFTStrategy):

    def wrap_model(self, method, config):
        # 冻结 Vision 和 LLM
        for p in method.vision.parameters():
            p.requires_grad = False
        for p in method.llm.parameters():
            p.requires_grad = False
        # 只解冻 Projector
        for p in method.projector.parameters():
            p.requires_grad = True

    def trainable_params(self, method):
        return [{"params": list(method.projector.parameters())}]

    def save_adapter(self, method, path):
        torch.save(method.projector.state_dict(), path + "/projector.pt")

    def load_adapter(self, method, path):
        method.projector.load_state_dict(torch.load(path + "/projector.pt"))

# 注册 + 使用
registry.register("peft", "projector_only", ProjectorOnlyStrategy)

config = TrainingConfig(
    peft=PEFTConfig(name="projector_only"),
    learning_rate=1e-3,   # Projector 用较大的 lr
    num_epochs=1,
)

LoRA-in-LoRA — 嵌套 LoRA 持续学习

类比：你有一本教科书（原始权重），上面贴了一张大便签（外层 LoRA，记住旧任务）。现在学新任务时，你在大便签上面再贴一张小便签（内层 LoRA）。学完后把小便签的内容合并进大便签。

原理

问题：直接用 LoRA 学新任务会忘记旧任务（灾难性遗忘）。

方案：两层 LoRA 嵌套结构：

• 外层 LoRA (A_out, B_out)：存储所有已学任务的知识。学习率小，慢慢积累。
• 内层 LoRA (A_in, B_in)：只负责学习当前新任务。学习率大，快速适应。
• 每个任务结束后，merge 内层 → 外层，然后重置内层。

y = W·x + B_out·A_out·x + B_in·A_in·x

在 mmit 中使用

class LoRAInLoRAStrategy(PEFTStrategy):

    def wrap_model(self, method, config):
        inner_rank = config.extra.get("inner_rank", 4)
        # 注入双层 LoRA 到 method.llm
        # 外层 rank=16, 内层 rank=4
        ...

    def trainable_params(self, method):
        return [
            {"params": outer_params, "lr": 1e-5},  # 外层：小 lr 保持旧知识
            {"params": inner_params, "lr": 1e-4},  # 内层：大 lr 学新任务
        ]

    def on_task_start(self, method, task_id, config):
        # 新任务开始：重新初始化内层 LoRA
        self._reinit_inner(method.llm)

    def on_task_end(self, method, task_id, config):
        # 任务结束：把内层 merge 进外层
        self._merge_inner_to_outer(method.llm)

    ...

registry.register("peft", "lora-in-lora", LoRAInLoRAStrategy)

# 持续学习：多任务顺序训练
trainer = Trainer.from_config(method, config)
trainer.train_continual([task1, task2, task3])

MoReS / LLaVA Steering — 模态表示转向

类比：想象 embedding 空间是一片海洋，不同模态（视觉/文本）的特征像船一样在海上漂浮。Steering 就是给每条船加一个小推进器，把它推向更好的位置，让不同模态的船靠得更近。

原理

与 LoRA 的根本区别：MoReS 不修改模型权重！

它在 embedding 空间中学习一个方向向量 (steering vector)，在 forward 时加到特定模态的 token embedding 上：

h' = h + α · v_steer

v_steer 是一个可学习的向量，α 是缩放因子。
效果：把视觉特征在 embedding 空间中"推向"更好的方向，改善模态对齐。

在 mmit 中使用

class MoReSStrategy(PEFTStrategy):

    def wrap_model(self, method, config):
        steer_dim = config.extra.get("steer_dim", 4096)
        alpha = config.extra.get("alpha", 0.1)

        # 创建可学习的 steering vector
        method._steer_vector = nn.Parameter(torch.randn(steer_dim) * 0.01)
        method._steer_alpha = alpha

        # 冻结所有其他参数
        for p in method.parameters():
            if p is not method._steer_vector:
                p.requires_grad = False

        # Hook: 在 forward 时加上 steering
        # method.forward_loss 中在 projector 输出后
        # 加上 steer_alpha * steer_vector

    def trainable_params(self, method):
        return [{"params": [method._steer_vector]}]
        # 只有一个向量！参数量 = 4096 ≈ 16KB

    def save_adapter(self, method, path):
        torch.save(method._steer_vector, path + "/steer.pt")

    ...

registry.register("peft", "mores", MoReSStrategy)

config = TrainingConfig(
    peft=PEFTConfig(name="mores", extra={"alpha": 0.1}),
)

Adapter Layer — 串联 Bottleneck

类比：在流水线的每两道工序之间，加一个"质检+微调"小站。产品经过小站时被微调一下，但整体流水线不变。

原理

在 Transformer 每一层的FFN 后面串联一个小的 bottleneck 网络：

adapter(x) = x + Up(ReLU(Down(x)))

• Down: d → r（降维，如 4096 → 64）
• ReLU: 非线性激活
• Up: r → d（升维，如 64 → 4096）
• 残差连接：加回原始输入

r 越小，参数越少（LaVIN 的 RepAdapter 甚至用 r=8）。

在 mmit 中使用

class AdapterLayerStrategy(PEFTStrategy):

    def wrap_model(self, method, config):
        bottleneck = config.extra.get("bottleneck", 64)

        # 冻结 LLM 所有参数
        for p in method.llm.parameters():
            p.requires_grad = False

        # 在每个 Transformer 层后面插入 adapter
        for layer in method.llm.model.layers:
            d = layer.mlp.down_proj.out_features
            adapter = nn.Sequential(
                nn.Linear(d, bottleneck),
                nn.ReLU(),
                nn.Linear(bottleneck, d),
            )
            layer._adapter = adapter  # 挂载

        # 修改 forward 使数据流经 adapter
        ...

    def trainable_params(self, method):
        params = []
        for layer in method.llm.model.layers:
            params.extend(layer._adapter.parameters())
        return [{"params": params}]

    ...

registry.register("peft", "adapter_layer", AdapterLayerStrategy)

config = TrainingConfig(
    peft=PEFTConfig(name="adapter_layer", extra={"bottleneck": 64}),
)
对比 LoRA：Adapter Layer 是"串联"在数据流中，数据必须经过它，所以推理时会有额外延迟。而 LoRA 是"并联"的，训练完可以 merge 回原权重，推理无额外开销。

Full Fine-tune — 全量微调

类比：不加任何便签，直接在教科书上改字。效果可能最好，但要小心：改太多可能把原来的好内容也改坏了。

原理

不使用任何 adapter，直接训练模型的所有参数（或指定的组件）。

优点：理论上表达能力最强
缺点：显存需求大、容易过拟合/遗忘、每个任务需要存一份完整模型

适用于：数据量大 + 计算资源充足 + 不需要保留原始模型能力的场景。

在 mmit 中使用

class FullFinetuneStrategy(PEFTStrategy):

    def wrap_model(self, method, config):
        components = config.extra.get("components", ["llm"])
        # 冻结所有
        for p in method.llm.parameters(): p.requires_grad = False
        for p in method.projector.parameters(): p.requires_grad = False
        for p in method.vision.parameters(): p.requires_grad = False

        # 解冻指定组件
        if "llm" in components:
            for p in method.llm.parameters(): p.requires_grad = True
        if "projector" in components:
            for p in method.projector.parameters(): p.requires_grad = True
        if "vision" in components:
            for p in method.vision.parameters(): p.requires_grad = True

    def trainable_params(self, method):
        return [{"params": [p for p in method.parameters() if p.requires_grad]}]

    def save_adapter(self, method, path):
        # 保存整个模型（因为没有单独的 adapter）
        method.llm.save_pretrained(path)

    def load_adapter(self, method, path):
        method.llm = method.llm.from_pretrained(path)

registry.register("peft", "full_finetune", FullFinetuneStrategy)

config = TrainingConfig(
    peft=PEFTConfig(name="full_finetune",
                    extra={"components": ["llm", "projector"]}),
    learning_rate=2e-5,
)
注意：虽然叫 "Full Fine-tune"，在 mmit 框架里它仍然是一个 PEFTStrategy 的子类。框架不关心策略里面是不是"参数高效" — 它只关心你实现了 4 个接口方法。这就是抽象的力量。
