Part 6: 反向传播 — 梯度计算（完整过程）

这是最难的部分。目标：算出 ∂L/∂W 和 ∂L/∂b，让 Part 7 能更新参数。

反向传播 = 从 loss 出发，沿着前向传播的路径倒着走，用链式法则逐层算梯度。

前向路径回顾：
h₀ → 线性层 (W, b) → h₁ → 输出头 (H) → logits → softmax → probs → CE loss → loss

反向路径（倒着走）：
loss → ∂L/∂logits (Step 6a) → ∂L/∂h₁ (Step 6b) → ∂L/∂W, ∂L/∂b (Step 6c)

Step 6a: ∂L/∂logits — 从 loss 反推到 logits 的梯度

公式：Softmax + CE 组合的梯度有一个非常简洁的结果：

∂L/∂logits = (softmax 输出 - one_hot(正确答案)) / batch_size

为什么除以 batch_size？因为 total_loss = (loss₀ + loss₁) / 2，除以 2 取平均，所以梯度也要除以 2。

位置 0（目标 = 坐, ID=1）：

softmax 输出（Part 4 的结果）= [P(猫), P(坐), P(了)] = [0.3316, 0.3441, 0.3244]
one_hot(坐=1) = [0, 1, 0]

grad_logits[0] = ([0.3316, 0.3441, 0.3244] - [0, 1, 0]) / 2
  = [0.3316-0, 0.3441-1, 0.3244-0] / 2
  = [0.3316, -0.6559, 0.3244] / 2
  = [0.1658, -0.3280, 0.1622]

位置 1（目标 = 了, ID=2）：

softmax 输出 = [0.3089, 0.3716, 0.3195]
one_hot(了=2) = [0, 0, 1]

grad_logits[1] = ([0.3089, 0.3716, 0.3195] - [0, 0, 1]) / 2
  = [0.3089, 0.3716, -0.6805] / 2
  = [0.1545, 0.1858, -0.3402]
直觉：梯度的正负很有意义。位置 0 中，坐 的梯度是 -0.3280（负数 = 需要增大这个 logit），猫 和 了 的梯度是正数（需要减小）。模型说"坐的分数不够高，猫和了的分数太高"，梯度指向修正的方向。

Step 6b: ∂L/∂h₁ — 从 logits 反推到 h₁ 的梯度

前向是 logits = h₁ 和 H 每行做点积。反向是 grad_h₁ = grad_logits @ H.weight。

H.weight (3×2) = [[0.4, -0.3], [0.1, 0.5], [-0.2, 0.1]]
（3 行分别是猫、坐、了的"判别向量"）

位置 0：grad_logits[0] (1×3) @ H.weight (3×2) → (1×2)

grad_h₁[0][第0维] = 0.1658×0.4 + (-0.3280)×0.1 + 0.1622×(-0.2)
               = 0.0663 - 0.0328 - 0.0324 = 0.0011

grad_h₁[0][第1维] = 0.1658×(-0.3) + (-0.3280)×0.5 + 0.1622×0.1
               = -0.0497 - 0.1640 + 0.0162 = -0.1975

∴ grad_h₁[0] = [0.0011, -0.1975]

位置 1：grad_logits[1] (1×3) @ H.weight (3×2) → (1×2)

grad_h₁[1][第0维] = 0.1545×0.4 + 0.1858×0.1 + (-0.3402)×(-0.2)
               = 0.0618 + 0.0186 + 0.0680 = 0.1484

grad_h₁[1][第1维] = 0.1545×(-0.3) + 0.1858×0.5 + (-0.3402)×0.1
               = -0.0464 + 0.0929 - 0.0340 = 0.0125

∴ grad_h₁[1] = [0.1484, 0.0125]
规律：矩阵乘法的反向传播有一个统一规则——
前向：output = input @ WeightT
反向：grad_input = grad_output @ Weight（不转置）
前向转置了，反向就不转置。对称的。

Step 6c: ∂L/∂W 和 ∂L/∂b — 我们要更新的参数的梯度

前向是 h₁ = h₀ @ WT + b。现在要算 W 和 b 的梯度。

∂L/∂b（偏置的梯度）：最简单——直接把所有位置的 grad_h₁ 加起来。

∂L/∂b = grad_h₁[0] + grad_h₁[1]
      = [0.0011, -0.1975] + [0.1484, 0.0125]
      = [0.1495, -0.1850]
为什么 b 的梯度这么简单？因为前向时 b 被直接加到每个位置上（h₁ = ... + b），所以反向时梯度就是直接传过来，所有位置求和。

∂L/∂W（权重的梯度）：需要矩阵乘法。公式：

∂L/∂W[i, j] = Σpos grad_h₁[pos, i] × h₀[pos, j]

即：W 的梯度 = grad_h₁ 的转置 @ h₀。shape: (2×2)ᵀ @ (2×2) = (2×2)，和 W 一样。

回顾 h₀ = [[0.1, 0.2], [0.3, 0.4]]（Part 1 的 Embedding 结果）。

∂L/∂W[0,0] = grad_h₁[pos0, 0] × h₀[pos0, 0] + grad_h₁[pos1, 0] × h₀[pos1, 0]
           = 0.0011 × 0.1 + 0.1484 × 0.3
           = 0.0001 + 0.0445 = 0.0446

∂L/∂W[0,1] = 0.0011 × 0.2 + 0.1484 × 0.4
           = 0.0002 + 0.0594 = 0.0596

∂L/∂W[1,0] = (-0.1975) × 0.1 + 0.0125 × 0.3
           = -0.0198 + 0.0038 = -0.0160

∂L/∂W[1,1] = (-0.1975) × 0.2 + 0.0125 × 0.4
           = -0.0395 + 0.0050 = -0.0345

∴ ∂L/∂W = [[0.0446, 0.0596], [-0.0160, -0.0345]]
直觉：∂L/∂W 告诉你"W 的每个元素应该往哪个方向调、调多少"。
正梯度 → 需要减小（当前值太大了导致 loss 高）
负梯度 → 需要增大（当前值太小了）
数值越大 → 需要调整的幅度越大

在 Part 7，SGD 会做 W_new = W - lr × ∂L/∂W，正好把梯度方向"翻转"过来去减小 loss。
PyTorch: loss.backward()   # 一行代码自动完成以上所有计算！梯度存在 W.grad 和 b.grad 中
回顾：你刚才做了什么？
Step 6a: loss → ∂L/∂logits（用 softmax - one_hot 公式）
Step 6b: ∂L/∂logits → ∂L/∂h₁（反向过输出头 H）
Step 6c: ∂L/∂h₁ → ∂L/∂W, ∂L/∂b（反向过线性层）

这就是链式法则：从最后一层开始，逐层向前传递梯度。PyTorch 的 autograd 引擎自动做这件事，你永远不需要手算。但理解了这个过程，你就知道 loss.backward() 内部在干什么。
