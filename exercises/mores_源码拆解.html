Part 1: OrthogonalIntervention.__init__ (Line 42-70) — 创建干预模块

源代码
class OrthogonalIntervention(nn.Module):
    """Low-rank orthogonal intervention module.
    Implements: h' = h + R^T * (W*h + b - R*h)
    """

    def __init__(self, hidden_dim: int,
                 rank: int = 1,
                 dropout: float = 0.0):
        super().__init__()
        self.hidden_dim = hidden_dim
        self.rank = rank

        # R: orthogonal projection [rank, hidden_dim]
        self.R = nn.Parameter(
            torch.empty(rank, hidden_dim))
        nn.init.orthogonal_(self.R)

        # W: steering matrix [rank, hidden_dim]
        self.W = nn.Linear(hidden_dim, rank, bias=True)
        nn.init.zeros_(self.W.weight)
        nn.init.zeros_(self.W.bias)

        self.dropout = nn.Dropout(dropout) \
            if dropout > 0 \
            else nn.Identity()

Part 2: orthogonal_penalty (Line 72-76) — 正交性惩罚

源代码
def orthogonal_penalty(self) -> torch.Tensor:
    """Soft orthogonality: ||R*R^T - I||_F^2"""
    RRt = self.R @ self.R.t()
    I = torch.eye(self.rank,
        device=self.R.device,
        dtype=self.R.dtype)
    return (RRt - I).pow(2).sum()
具体计算过程 (rank=1, hidden_dim=4096)
第一步: R @ R.t() 
R shape:    (1, 4096)
R.t() shape: (4096, 1)
RRt = R @ R.t()
    = (1, 4096) @ (4096, 1)
    = (1, 1)
    = [[s]]   其中 s = ||R||^2 (R 的 L2 范数平方)
第二步: I = eye(1) = [[1]]
情况 1: R 是单位向量 (||R||=1) 
s = 1.0, RRt = [[1.0]]
penalty = (1.0 - 1.0)^2 = 0.0 ← 完美!
情况 2: R 漂移了 (||R||=1.05) 
s = 1.1025, RRt = [[1.1025]]
penalty = (1.1025 - 1.0)^2 = 0.0105
情况 3: R 严重漂移 (||R||=2.0) 
s = 4.0, RRt = [[4.0]]
penalty = (4.0 - 1.0)^2 = 9.0 ← 很大的惩罚!
用途：这个惩罚会乘以 0.01 加到 loss 里。它迫使 R 始终保持为正交矩阵（rank=1 时就是单位向量）。如果不加这个约束，R 可以任意缩放，干预的几何意义就丢了。

Part 3: forward — 简单路径 (Line 98-105) — 干预所有 token

源代码
def forward(self, h, vis_mask=None,
            steer_ratio=1.0):
    R = self.R  # [rank, hidden_dim]

    if vis_mask is None and \
       steer_ratio >= 1.0:
        # Steer all tokens (simple path)
        Rh = F.linear(h, R)
        # [B, seq, rank]
        Wh = self.W(h)
        # [B, seq, rank]
        delta_low = self.dropout(Wh - Rh)
        delta = F.linear(delta_low, R.t())
        return h + delta
逐步计算 (batch=1, seq=620, dim=4096)
输入: 
h: (1, 620, 4096)
  ← 1 个 batch, 620 个 token, 4096 维
R: (1, 4096)  ← 正交投影方向
Step 1: Rh = F.linear(h, R) 
F.linear(h, R) = h @ R.T
  h:   (1, 620, 4096)
  R.T: (4096, 1)
  Rh:  (1, 620, 1)
← 每个 token 投影到 R 方向上的分量（标量）
Step 2: Wh = self.W(h) 
W 是 Linear(4096, 1):
  h:  (1, 620, 4096)
  Wh: (1, 620, 1)
← 每个 token 经过 W 变换后的标量
Step 3: delta_low = dropout(Wh - Rh) 
Wh - Rh: (1, 620, 1) - (1, 620, 1)
        = (1, 620, 1)
← 低维空间中的"修正量"
Step 4: delta = F.linear(delta_low, R.t()) 
F.linear(delta_low, R.t()) = delta_low @ R
  delta_low: (1, 620, 1)
  R:         (1, 4096)
  delta:     (1, 620, 4096)
← 把低维修正投射回 4096 维空间
Step 5: return h + delta 
h + delta: (1, 620, 4096) + (1, 620, 4096)
         = (1, 620, 4096)  ← 形状不变!
初始化时 W=0 意味着什么？ 
训练开始时 W.weight=0, W.bias=0, 所以:
  Wh = 0
  delta_low = 0 - Rh = -Rh
  delta = -R^T @ R @ h
  h' = h - R^T(R @ h)

这就是投影去除! 把 h 在 R 方向上的分量减掉。
= 从每个 token 中移除 R 所指方向的信息。

随着训练，W 学到非零值 → 变成"先移除旧信息，再注入新信息"。

Part 4: forward — 选择性路径 (Line 107-135) — 只干预 top 1% 视觉 token

源代码
# ── Selective steering ──
h_out = h.clone()

for b in range(h.size(0)):
    if vis_mask is not None:
        vis_idx = vis_mask[b].nonzero(
            as_tuple=True)[0]
    else:
        vis_idx = torch.arange(
            h.size(1), device=h.device)

    if vis_idx.numel() == 0:
        continue

    # Select top-k% by norm
    if steer_ratio < 1.0 \
       and vis_idx.numel() > 1:
        h_vis = h[b, vis_idx]
        norms = h_vis.norm(dim=-1)
        k = max(1,
            int(vis_idx.numel()
                 * steer_ratio))
        _, topk_local = norms.topk(k)
        vis_idx = vis_idx[topk_local]

    # Apply intervention
    h_sel = h[b, vis_idx]
    Rh = F.linear(h_sel, R)
    Wh = self.W(h_sel)
    delta_low = self.dropout(Wh - Rh)
    delta = F.linear(delta_low, R.t())
    h_out[b, vis_idx] = h_sel + delta

return h_out

              ↑文本  ↑-------620个视觉------↑  ↑文本

        = tensor([2, 3, 4, ..., 621])
        → 620 个视觉 token 的索引
Step 3: 按 L2 范数选 top 1% 
h_vis = h[0, vis_idx]  → (620, 4096)
norms = h_vis.norm(dim=-1) → (620,)
  例如: [12.3, 8.7, 15.1, 9.4, ...]

k = max(1, int(620 * 0.01)) = max(1, 6) = 6

_, topk_local = norms.topk(6)
  topk_local = [2, 45, 103, 287, 412, 599]
  ← 范数最大的 6 个 token 的局部索引

vis_idx = vis_idx[topk_local]
  ← 转回全局索引，例如 [4, 47, 105, 289, 414, 601]
Step 4: 只对 6 个 token 做干预

← 只修改这 6 个位置，其余 token 不变!
为什么只选 top 1%？论文发现：范数最大的视觉 token 包含最关键的视觉信息。只干预这些 token，效果反而更好（论文 Table 3）。而且计算量极小: 620 个 token 中只处理 6 个。

Part 5: 辅助函数 (Line 138-234) — 解析层位置 + 查找模型结构

def _parse_positions(positions,
                     num_layers: int):
    """解析层位置。接受多种格式:
    "f4+l5" → 前4 + 后5
    "all"   → 全部
    "f4"    → 前4
    "l5"    → 后5
    """
    if isinstance(positions, list):
        return sorted(idx for idx
            in positions
            if 0 <= idx < num_layers)

    position_str = str(positions).\
        strip().lower()
    if position_str == "all":
        return list(range(num_layers))

    layers = set()
    for part in position_str.split("+"):
        match = re.match(
            r"^([fl])(\d+)$", part)
        if match:
            prefix = match.group(1)
            count = int(match.group(2))
            if prefix == "f":
                layers.update(
                    range(min(count,
                        num_layers)))
            else:
                layers.update(
                    range(max(0,
                        num_layers - count),
                        num_layers))
    return sorted(layers)

Part 6c: _prepare_model_impl — 注册 hooks (Line 336-375) — 核心机制

源代码
# Attach hooks
method_ref = self

for idx, iv in zip(target_indices,
                    interventions):
    def make_hook(intervention):
        def hook(module, input, output):
            if isinstance(output, tuple):
                h = output[0]
            else:
                h = output

            # Build visual token mask
            vis_mask = None
            if method_ref._vis_only \
               and method_ref.\
                   _image_token_id \
                   is not None:
                ids = getattr(model,
                    "_mores_input_ids",
                    None)
                if ids is not None \
                   and ids.shape[1] \
                       == h.shape[1]:
                    vis_mask = \
                        (ids == method_ref.\
                             _image_token_id)

            h_new = intervention(
                h, vis_mask=vis_mask,
                steer_ratio=\
                    method_ref._steer_ratio)

            if isinstance(output, tuple):
                return (h_new,) + output[1:]
            return h_new
        return hook

    h = layer_list[idx].\
        register_forward_hook(
            make_hook(iv))
    self._hooks.append(h)
内部发生了什么
为什么需要 make_hook 闭包工厂？ 
如果直接写:
  for idx, iv in zip(...):
    def hook(module, input, output):
        intervention(h, ...)  ← 捕获的是变量 iv

问题: Python 闭包捕获的是变量不是值!
循环结束后 iv 指向最后一个元素，
所有 9 个 hook 都会用同一个 iv。

make_hook(intervention) 创建新的作用域:
  intervention 是函数参数 → 每次调用绑定不同的值
  (虽然 share_weights 时值相同，但这是正确的模式)
注册结果： 
layer_list[0]._forward_hooks = {
  hook_id_0: <hook 函数>  ← 新增!
}
layer_list[1]._forward_hooks = {
  hook_id_1: <hook 函数>  ← 新增!
}
...
layer_list[31]._forward_hooks = {
  hook_id_8: <hook 函数>  ← 新增!
}

共 9 个 hook，绑定到 9 层:
  层 0,1,2,3 和 层 27,28,29,30,31
hook 的执行时机： 
当 model(x) 执行时:
  layer_0.forward(x) 正常执行
  → 输出 output
  → PyTorch 自动调用 _forward_hooks 中的函数
  → hook(layer_0, input, output)
  → output 被替换为 h + delta
  → 传给 layer_1

关键: LLM 权重完全不变，
hook 在"层的输出"上动手脚。

内部发生了什么
这是 pre-hook（在 forward 之前执行）： 
model._forward_pre_hooks = {
  pre_hook_id: _capture_input_ids  ← 新增!
}

执行顺序:
1. model(input_ids=..., images=...)
2. _capture_input_ids 先执行
   → model._mores_input_ids = input_ids
   例如: tensor([[1, 345, 32000, 32000, ..., 2]])
                          ↑ image token = 32000
3. model.forward() 正常执行
4. 每一层的 hook 读取 _mores_input_ids
   → vis_mask = (ids == 32000)
   → tensor([[F, F, T, T, ..., F]])
为什么需要 pre-hook？forward_hook 只能访问当前层的 input/output。但 input_ids 只在模型最顶层可见（被 embedding 层转换后就消失了）。所以用 pre-hook 把 input_ids 存到 model 上，供各层 hook 读取。

Part 7: compute_loss (Line 409-423) — 计算训练损失

源代码
def compute_loss(self, model,
                 batch, outputs):
    text_loss = _standard_ce_loss(
        outputs, batch)
    metrics = {
        "text_loss": text_loss.item()}

    # Soft orthogonality penalty on R
    ortho_loss = torch.tensor(
        0.0, device=text_loss.device,
        dtype=text_loss.dtype)
    for iv in self._all_interventions:
        ortho_loss = ortho_loss \
            + iv.orthogonal_penalty()
    if ortho_loss.item() > 0:
        total_loss = text_loss \
            + 0.01 * ortho_loss
        metrics["ortho_loss"] = \
            ortho_loss.item()
    else:
        total_loss = text_loss

    return total_loss, metrics
具体计算
两部分损失： 
1. text_loss = CrossEntropyLoss(logits, labels)
   ← 和 LoRA、Full FT 完全一样的文本生成损失
   ← 衡量模型生成回答的质量

2. ortho_loss = intervention.orthogonal_penalty()
   ← ||R @ R.T - I||^2
   ← 迫使 R 保持正交
训练中的实际数值： 
text_loss  = 2.3400  ← CE loss
ortho_loss = 0.0020  ← R 几乎是正交的

total = 2.3400 + 0.01 * 0.0020
      = 2.3400 + 0.00002
      = 2.34002

正交惩罚的贡献非常小 (0.001%),
但它是必要的 — 没有它 R 会渐渐偏离正交，
干预的几何意义会退化。
梯度只流向 8,193 个参数：虽然 forward 经过了 7B 个参数，但因为 requires_grad=False，backward 只计算 R, W.weight, W.bias 的梯度。优化器也只更新这 3 个张量。

            f"intervention_{i}"] = \
            iv.state_dict()
    torch.save(intervention_states,
        os.path.join(path,
            "mores_interventions.pt"))

    # Save projector weights
    projector_state = {}
    for name, p in \
        model.named_parameters():
        if "projector" in name.lower():
            projector_state[name] = p.data
    if projector_state:
        torch.save(projector_state, ...)

    processor.save_pretrained(path)
    # + save metadata JSON
保存了什么
intervention.state_dict() 的内容： 
{
  "R":        tensor(1, 4096),   ← 4096 floats
  "W.weight": tensor(1, 4096),   ← 4096 floats
  "W.bias":   tensor(1),         ← 1 float
}

只有 3 个张量，共 8,193 个 float16 值。
文件大小对比：
方法保存文件大小内容
Full FT~14 GB整个模型所有参数
LoRA (rank=128)~100 MBA, B 矩阵 (每层)
MoReS (rank=1)~32 KBR + W.weight + W.bias
32 KB = 8193 * 2 bytes (float16) + 文件头
      ≈ 16,386 bytes ≈ 16 KB
加上 metadata JSON ≈ ~32 KB 总计
32 KB vs 14 GB:差了 400,000 倍。你可以用微信发送 MoReS 的 checkpoint。

加载的完整流程
三步加载： 
Step 1: 加载原始 LLaVA-1.5-7B (~14GB)
  ← 和预训练模型完全一样

Step 2: prepare_model(model, ...)
  ← 冻结所有参数
  ← 创建新的 OrthogonalIntervention (随机初始化)
  ← 注册 9 个 forward hooks
  ← 注册 1 个 pre-hook
  此时: intervention 参数是随机的

Step 3: iv.load_state_dict(states[...])
  ← 从 32KB 文件读取训练好的 R, W.weight, W.bias
  ← 覆盖随机初始化的值
  此时: intervention 参数是训练好的

Step 4: model.eval()
  ← dropout 关闭
  ← 准备推理
加载速度：主要时间花在 Step 1 加载 7B 基座模型。Step 3 加载 32KB 几乎瞬间完成。
