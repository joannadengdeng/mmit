Step 6: model(x) — __call__ 的完整执行流程

你写的代码
# forward 定义
def forward(self, x):
  h = self.embed(x)
  h = self.linear(h)
  h = h + F.linear(h, self.R) * self.scale
  return h

# 调用
x = torch.tensor([0, 1, 2])
output = model(x)
# ↑ 是 model(x) 不是 model.forward(x)!
model(x) 内部发生了什么
model(x) = nn.Module.__call__(x)： 
def __call__(self, *args):

  ① 运行 pre_hooks
  for hook in self._forward_pre_hooks:
      hook(self, args)
  → 目前为空，跳过

  ② 调用 self.forward(x)
  → self.embed(x)
      → Embedding.__call__(x)
        → embed.forward: 查表
        → 返回 (3, 4) tensor
  → self.linear(h)
      → Linear.__call__(h)
        → linear.forward: h @ W.T + b
        → 返回 (3, 4) tensor
  → F.linear(h, self.R)
      → 纯函数，不触发 __call__
      → h @ R.T → (3, 1)

  ③ 运行 post_hooks
  → 目前为空，跳过

  return result
每一步的实际 shape： 
x:                     (3,)     int64
embed(x):              (3, 4)   float32
linear(h):             (3, 4)   float32
F.linear(h, self.R):   (3, 1)   float32
h + (...) * 2.0:       (3, 4)   float32
output:                (3, 4)

Step 7: register_forward_hook — 不改代码就修改行为

你写的代码
# 定义 hook
def my_hook(module, input, output):
    print(f"Hook! output[0,0]="
          f"{output[0,0]:.4f}")
    return output + 0.01

# 注册到 linear 层
handle = model.linear \
  .register_forward_hook(my_hook)

# 调用模型
output = model(x)

# 移除 hook
handle.remove()

# 再调一次
output2 = model(x)
# ← 没有 print → hook 已经不在了
内部状态变化 + 实际输出
注册后： 
model.linear._forward_hooks = {
  0: my_hook  ← 新增！
}
model(x) 时的 print 输出： 
Hook! output[0,0]=0.2847

→ linear.__call__(h) 执行时：
  ① linear.forward(h) → 原始输出
  ② my_hook(linear, (h,), 原始输出)
     → print 被调用
     → return output + 0.01
  ③ 后续计算用的是 +0.01 后的值
handle.remove() 后： 
model.linear._forward_hooks = {}  ← 清空了
再次 model(x)： 
（没有 print 输出）
→ hook 已不存在，output 是原始值
MoReS 就是这样工作的：在 9 个 LLaMA 层注册 hook，hook 内部做 h'=h+R^T(Wh-Rh)。模型代码一行不改。

附录：nn.Module 关键方法的简化源码

下面是 nn.Module 中你会间接调用到的核心方法。这些方法你不需要自己写，但理解它们才能明白"为什么 self.linear = nn.Linear(...) 就能被自动追踪"。

__setattr__ — 赋值时自动分拣

每次你写 self.xxx = yyy，Python 不是直接存，而是先调这个方法。nn.Module 重写了它来做"自动分拣"。

简化源码
# torch/nn/modules/module.py 简化版

def __setattr__(self, name, value):

  # 情况 1: 赋值的是 Parameter
  if isinstance(value, Parameter):
    # 从 _modules 里删掉同名的（如果有）
    self._modules.pop(name, None)
    # 存入 _parameters 字典
    self._parameters[name] = value

  # 情况 2: 赋值的是 Module
  elif isinstance(value, Module):
    # 从 _parameters 里删掉同名的
    self._parameters.pop(name, None)
    # 存入 _modules 字典
    self._modules[name] = value

  # 情况 3: 其他（int, float, tensor, ...）
  else:
    # 当普通 Python 属性存储
    # 不进任何字典！
    object.__setattr__(self, name, value)
触发场景 + 实际效果
你写的代码 → 触发的分支 → 结果： 
self.linear = nn.Linear(4,4)
  → 情况 2: isinstance(Linear, Module) = True
  → self._modules["linear"] = Linear对象
  → 被 .parameters() 找到 ✓

self.R = nn.Parameter(tensor)
  → 情况 1: isinstance(Parameter) = True
  → self._parameters["R"] = Parameter对象
  → 被 .parameters() 找到 ✓

self.scale = 2.0
  → 情况 3: float 既不是 Module 也不是 Parameter
  → object.__setattr__(self, "scale", 2.0)
  → 普通属性，不被追踪 ✗

self.mask = torch.ones(3,3)
  → 情况 3: Tensor 不是 Parameter！
  → 普通属性，不被追踪 ✗
  → 要追踪就用 nn.Parameter(tensor)
    或 self.register_buffer("mask", tensor)

__call__ — model(x) 的真正入口

你写 model(x) 时调的不是 forward，而是这个方法。它在 forward 前后插入 hook。

简化源码
def __call__(self, *args, **kwargs):

  # ① 运行所有 pre_hook
  for hook in self._forward_pre_hooks.values():
    result = hook(self, args)
    if result is not None:
      args = result

  # ② 调用你写的 forward
  result = self.forward(*args, **kwargs)

  # ③ 运行所有 post_hook
  for hook in self._forward_hooks.values():
    hook_result = hook(self, args, result)
    if hook_result is not None:
      result = hook_result  # 替换输出！

  return result
执行流程示例
model(x) 时实际发生的事： 
model(x)
  │
  ├─ ① _forward_pre_hooks
  │    MoReS 用这个捕获 input_ids:
  │    hook 把 input_ids 存到 model 上
  │    以便后面的 post_hook 使用
  │
  ├─ ② self.forward(x)
  │    你写的前向逻辑
  │    内部调用 self.embed(x) 时
  │    又触发 Embedding.__call__(x)
  │    → 同样的 pre_hook/forward/post_hook
  │
  └─ ③ _forward_hooks
       MoReS 的干预在这里：
       hook 拿到 forward 的 output
       做 h' = h + R^T(Wh - Rh)
       return 修改后的 h'
       → 替换原始 output
所以 model(x) ≠ model.forward(x) ! 直接调 forward 跳过了 hook。

parameters() / named_parameters() — 递归遍历所有参数

简化源码
def named_parameters(self, prefix=''):
  # 第一步: 自己的 _parameters
  for name, param in self._parameters.items():
    yield prefix + name, param

  # 第二步: 递归遍历每个子模块
  for mod_name, module in self._modules.items():
    subprefix = prefix + mod_name + '.'
    # 递归！子模块也走同样的逻辑
    yield from module.named_parameters(subprefix)

def parameters(self):
  for _, param in self.named_parameters():
    yield param
遍历过程追踪
model.named_parameters() 的遍历路径： 
① self._parameters (prefix="")
   → yield "R", Parameter(1,4)

② self._modules["embed"] (prefix="embed.")
   → embed._parameters
   → yield "embed.weight", Parameter(3,4)

③ self._modules["linear"] (prefix="linear.")
   → linear._parameters
   → yield "linear.weight", Parameter(4,4)
   → yield "linear.bias", Parameter(4,)

结果:
  R              → shape (1, 4)
  embed.weight   → shape (3, 4)
  linear.weight  → shape (4, 4)
  linear.bias    → shape (4,)

注意: scale 和 mask 不在 _parameters
也不在 _modules 里 → 永远遍历不到

state_dict() / load_state_dict() — 保存和加载

简化源码
def state_dict(self, prefix=''):
  result = {}

  # 收集自己的 _parameters
  for name, param in self._parameters.items():
    result[prefix + name] = param.data

  # 收集自己的 _buffers
  for name, buf in self._buffers.items():
    result[prefix + name] = buf

  # 递归收集子模块
  for mod_name, module in self._modules.items():
    subprefix = prefix + mod_name + '.'
    result.update(module.state_dict(subprefix))

  return result

def load_state_dict(self, sd, strict=True):
  # 遍历 sd 的每个 key
  # 根据 "." 分割找到对应的子模块和参数
  # 把 sd[key] 的值复制到对应的 Parameter.data
  #
  # strict=True: key 不完全匹配就报错
  # strict=False: 只填匹配的，跳过不匹配的

to(device) — 递归移动所有参数到指定设备

简化源码
def to(self, *args, **kwargs):
  # 移动自己的 _parameters
  for name, param in self._parameters.items():
    self._parameters[name] = \
      Parameter(param.data.to(*args, **kwargs))

  # 移动自己的 _buffers
  for name, buf in self._buffers.items():
    self._buffers[name] = buf.to(*args, **kwargs)

  # 递归移动子模块
  for module in self._modules.values():
    module.to(*args, **kwargs)

  return self
实际效果
model.to("cuda") 移动路径：

self._buffers            → 空，跳过
self._modules["embed"]  → 递归
  embed._parameters["weight"].to("cuda") ✓
self._modules["linear"] → 递归
  linear weight, bias .to("cuda")       ✓

train() / eval() — 递归切换训练/推理模式

简化源码
def train(self, mode=True):
  self.training = mode
  for module in self._modules.values():
    module.train(mode)  # 递归
  return self

def eval(self):
  return self.train(False)

register_forward_hook() — 注册钩子

简化源码
def register_forward_hook(self, hook):
  # hook 签名: hook(module, input, output) -> output 或 None

  handle = RemovableHandle(self._forward_hooks)
  self._forward_hooks[handle.id] = hook

  return handle
  # handle.remove() 会从字典中删除这个 hook

def register_forward_pre_hook(self, hook):
  # hook 签名: hook(module, args) -> args 或 None

  handle = RemovableHandle(self._forward_pre_hooks)
  self._forward_pre_hooks[handle.id] = hook

  return handle
MoReS 的实际用法
MoReS 注册 hook 的代码（简化）： 
intervention = OrthogonalIntervention(4096, rank=1)

def make_hook(iv):
  def hook(module, input, output):
    h = output[0]
    h_new = iv(h)  # h' = h + R^T(Wh-Rh)
    return (h_new,) + output[1:]
  return hook

# 在第 0, 1, 2, 3, 27, 28, 29, 30, 31 层注册
for idx in [0,1,2,3,27,28,29,30,31]:
  model.layers[idx].register_forward_hook(
    make_hook(intervention)
  )
效果： 
model.layers[0]._forward_hooks = {0: hook}
model.layers[1]._forward_hooks = {1: hook}
...共 9 个层有 hook

model(x) 时，每个有 hook 的层：
  layer.forward(h) → 原始输出
  → hook(layer, (h,), 原始输出)
  → return 修改后的 h'

总览：nn.Module 的 10 个关键方法

方法你写的代码内部做了什么遍历哪个字典
__init__super().__init__()创建 _parameters, _modules 等 6 个空字典—
__setattr__self.xxx = yyy检测类型 → 自动存入对应字典_parameters 或 _modules
__call__model(x)pre_hooks → forward → post_hooks_forward_pre_hooks, _forward_hooks
forward你自己写的定义数据流—
parameters()model.parameters()递归收集所有 Parameter_parameters + 递归 _modules
state_dict()model.state_dict()递归收集参数值_parameters + _buffers + 递归 _modules
load_state_dict()model.load_state_dict(sd)按 key 填入参数值同上
to()model.to("cuda")递归移动到设备_parameters + _buffers + 递归 _modules
train()/eval()model.eval()递归设置 self.training递归 _modules
register_forward_hooklayer.register_forward_hook(fn)存入 _forward_hooks_forward_hooks
核心规律：几乎所有方法都是"遍历 _parameters + _buffers + 递归遍历 _modules"。理解了这个递归结构，就理解了 nn.Module 的一切。
