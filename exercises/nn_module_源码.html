nn.Module 源码拆解 — 它里面到底有什么

这份文档展示 nn.Module 类的完整简化源码。每个方法左边是源码，右边是"什么时候被调用 + 做了什么"。

基于 PyTorch 2.x 源码 (torch/nn/modules/module.py)，约 2000 行简化为 ~200 行核心逻辑。黑白打印。

Part 1: nn.Module 的全部属性（__init__ 创建的）

源码
class Module:

  def __init__(self):

    # ═══ 核心字典（一切魔法的基础）═══

    # 存 nn.Parameter 对象
    self._parameters = OrderedDict()

    # 存子 Module 对象
    self._modules = OrderedDict()

    # 存 register_buffer 注册的 tensor
    self._buffers = OrderedDict()

    # ═══ Hook 字典 ═══

    # forward 之后执行的 hook
    self._forward_hooks = OrderedDict()

    # forward 之前执行的 hook
    self._forward_pre_hooks = OrderedDict()

    # backward 时执行的 hook
    self._backward_hooks = OrderedDict()

    # ═══ 状态 ═══

    # True=训练模式, False=推理模式
    self.training = True
解释
7 个字典/属性 = nn.Module 的全部内部状态 
_parameters
  存什么: nn.Parameter 对象
  怎么存: self.R = nn.Parameter(...)
  谁遍历: parameters(), state_dict(), to()

_modules
  存什么: nn.Module 子类（Linear, Embedding等）
  怎么存: self.linear = nn.Linear(...)
  谁遍历: parameters()递归, state_dict()递归,
         to()递归, train()/eval()递归

_buffers
  存什么: 不需要训练但需要保存的 tensor
  怎么存: self.register_buffer("name", tensor)
  谁遍历: state_dict(), to()
  和 Parameter 的区别: 不出现在 parameters() 中
                     → optimizer 不会更新它

_forward_hooks
  存什么: forward 后要执行的函数
  怎么存: register_forward_hook(fn)
  谁调用: __call__() 在 forward 之后

_forward_pre_hooks
  同上，但在 forward 之前执行

_backward_hooks
  反向传播时执行（mmit 不用，了解即可）

training
  布尔值。Dropout 和 BatchNorm 根据它
  决定行为。train() 设为 True，eval() 设为 False。

Part 2: __setattr__ — 每次 self.xxx = yyy 都经过它

源码
def __setattr__(self, name, value):

  # 如果右边是 Parameter
  if isinstance(value, Parameter):
    # 清除同名的其他类型
    self._buffers.pop(name, None)
    self._modules.pop(name, None)
    # 存入 _parameters
    self._parameters[name] = value

  # 如果右边是 Module
  elif isinstance(value, Module):
    self._buffers.pop(name, None)
    self._parameters.pop(name, None)
    self._modules[name] = value

  # 其他情况
  else:
    # 如果之前这个名字在某个字典里
    # 先删掉（防止重复）
    if name in self._parameters:
      self._parameters.pop(name)
    if name in self._modules:
      self._modules.pop(name)
    if name in self._buffers:
      self._buffers.pop(name)
    # 当作普通 Python 属性
    object.__setattr__(self, name, value)
你写的代码 → 走哪个分支
self.linear = nn.Linear(4, 4)
  isinstance(Linear, Module) → True
  → self._modules["linear"] = Linear对象
  → 被 parameters() 递归找到 ✓

self.R = nn.Parameter(torch.randn(1,4))
  isinstance(Parameter) → True
  → self._parameters["R"] = Parameter对象
  → 被 parameters() 直接找到 ✓

self.scale = 2.0
  isinstance(float, Parameter) → False
  isinstance(float, Module) → False
  → object.__setattr__ → 普通属性
  → 不被任何方法找到 ✗

self.mask = torch.ones(3, 3)
  Tensor 不是 Parameter！也不是 Module！
  → object.__setattr__ → 普通属性
  → 不被追踪 ✗
  → .to("cuda") 不会移动它！

self.register_buffer("mask", torch.ones(3,3))
  这不走 __setattr__，而是直接操作:
  → self._buffers["mask"] = tensor
  → 被 state_dict() 保存 ✓
  → 被 to() 移动 ✓
  → 但不在 parameters() 里（不训练）

Part 3: __call__ — model(x) 的真正入口

源码
def __call__(self, *args, **kwargs):

  # ① pre hooks
  for hook in self._forward_pre_hooks.values():
    result = hook(self, args)
    if result is not None:
      if not isinstance(result, tuple):
        result = (result,)
      args = result

  # ② 你写的 forward
  result = self.forward(*args, **kwargs)

  # ③ post hooks
  for hook in self._forward_hooks.values():
    hook_result = hook(self, args, result)
    if hook_result is not None:
      result = hook_result

  return result
执行流程
model(x)  ← 你写的
  ↓
Module.__call__(model, x)  ← Python 实际调的
  │
  ├── ① self._forward_pre_hooks
  │   空 → 跳过
  │   MoReS: 存 input_ids 到 model 上
  │
  ├── ② self.forward(x)
  │   ← 你定义的前向逻辑
  │   内部每次 self.embed(x) 又触发
  │   Embedding.__call__(x) → 同样的流程
  │
  └── ③ self._forward_hooks
      空 → 跳过
      MoReS: 拿到 output
        做 h' = h + R^T(Wh-Rh)
        return h' → 替换 output
model(x) ≠ model.forward(x)
直接调 forward → 跳过 ①③ → hook 失效
MoReS/ReFT 的干预完全不执行！
永远写 model(x)，不写 model.forward(x)

Part 4: parameters() / named_parameters()

源码
def _named_members(self, get_members, prefix):
  """通用递归遍历"""
  # 已访问的模块（防重复）
  visited = set()

  for mod_name, module in self.named_modules():
    if module in visited:
      continue
    visited.add(module)
    members = get_members(module)
    for name, member in members:
      full_name = mod_name + '.' + name \
                  if mod_name else name
      yield full_name, member

def named_parameters(self):
  return self._named_members(
    lambda m: m._parameters.items(),
    prefix=''
  )

def parameters(self):
  for _, p in self.named_parameters():
    yield p

def named_modules(self):
  """递归返回自己和所有子模块"""
  yield '', self
  for name, mod in self._modules.items():
    for sub_name, sub_mod in mod.named_modules():
      full = name + '.' + sub_name \
             if sub_name else name
      yield full, sub_mod
遍历过程
named_modules() 遍历路径：
yield "", model              ← 自己
yield "embed", Embedding     ← 子模块
yield "linear", Linear       ← 子模块

named_parameters() 遍历路径：
对 model 自己:
  model._parameters → "R"
对 embed:
  embed._parameters → "embed.weight"
对 linear:
  linear._parameters
    → "linear.weight"
    → "linear.bias"

结果：
R              Parameter(1, 4)
embed.weight   Parameter(3, 4)
linear.weight  Parameter(4, 4)
linear.bias    Parameter(4,)

不含：
scale   ← 不在 _parameters 里
mask    ← 不在 _parameters 里

named_buffers() 同理，
但遍历的是 _buffers 而不是 _parameters。

Part 5: state_dict() / load_state_dict()

源码
def state_dict(self, prefix=''):
  result = OrderedDict()

  # 自己的参数
  for name, param in self._parameters.items():
    if param is not None:
      result[prefix + name] = param.data

  # 自己的 buffer
  for name, buf in self._buffers.items():
    if buf is not None:
      result[prefix + name] = buf

  # 递归子模块
  for name, mod in self._modules.items():
    if mod is not None:
      sub = mod.state_dict(prefix + name + '.')
      result.update(sub)

  return result

def load_state_dict(self, sd, strict=True):
  missing = []
  unexpected = []

  for key, value in sd.items():
    # 按 "." 分割 key 找到对应的子模块
    parts = key.split('.')
    mod = self
    for part in parts[:-1]:
      mod = mod._modules[part]
    param_name = parts[-1]

    if param_name in mod._parameters:
      mod._parameters[param_name].data = value
    elif param_name in mod._buffers:
      mod._buffers[param_name] = value
    else:
      unexpected.append(key)

  if strict and (missing or unexpected):
    raise RuntimeError(...)
实际效果
model.state_dict() 返回：
OrderedDict({
  "R":             tensor([[0.52, -0.81, 0.30, 0.15]]),
  "embed.weight":  tensor(3×4),
  "linear.weight": tensor(4×4),
  "linear.bias":   tensor(4,)
})

包含： _parameters 的 .data + _buffers
不含： scale(普通float), mask(普通tensor)

load_state_dict(sd) 匹配过程：
"R" → split(".") = ["R"]
  → mod = self
  → self._parameters["R"].data = sd["R"]

"embed.weight" → split(".") = ["embed", "weight"]
  → mod = self._modules["embed"]
  → mod._parameters["weight"].data = sd[...]

"linear.weight" → 同理
"linear.bias" → 同理

strict=True:
  sd 的 key 和 model 的 key 必须完全一致
  多了或少了 → RuntimeError

strict=False:
  只填匹配的，不匹配的静默跳过
  → Freeze Tuning 用这个
    （只保存了部分参数）

Part 6: to() / train() / eval()

源码
def to(self, *args, **kwargs):
  # 对自己的每个 parameter 和 buffer 做 .to()
  def convert(t):
    return t.to(*args, **kwargs)

  return self._apply(convert)

def _apply(self, fn):
  # 递归对所有子模块做
  for module in self._modules.values():
    module._apply(fn)

  # 对自己的 _parameters 做
  for key, param in self._parameters.items():
    self._parameters[key] = Parameter(
      fn(param.data), param.requires_grad
    )

  # 对自己的 _buffers 做
  for key, buf in self._buffers.items():
    self._buffers[key] = fn(buf)

  return self

def train(self, mode=True):
  self.training = mode
  for module in self._modules.values():
    module.train(mode)
  return self

def eval(self):
  return self.train(False)
实际效果
model.to("cuda") 的递归路径：

_apply(lambda t: t.to("cuda"))
  │
  ├── self._modules["embed"]._apply(fn)
  │     embed._parameters["weight"]
  │       = fn(weight) → weight.to("cuda") ✓
  │
  ├── self._modules["linear"]._apply(fn)
  │     linear._parameters["weight"] → cuda ✓
  │     linear._parameters["bias"]   → cuda ✓
  │
  ├── self._parameters["R"]
  │     = fn(R) → R.to("cuda")             ✓
  │
  └── self._buffers → 空，跳过

不移动的：
  self.scale = 2.0   → 不在任何字典
  self.mask = tensor  → 不在任何字典
  → mask 留在 CPU → device mismatch!

model.eval() 的递归路径：
  self.training = False
  → embed.training = False
  → linear.training = False
  所有子模块递归设为 False

Part 7: register_forward_hook / register_buffer

源码
def register_forward_hook(self, hook):
  # hook(module, input, output) -> output 或 None
  handle = RemovableHandle(
    self._forward_hooks
  )
  self._forward_hooks[handle.id] = hook
  return handle
  # handle.remove() 从字典中删除

def register_forward_pre_hook(self, hook):
  # hook(module, args) -> args 或 None
  handle = RemovableHandle(
    self._forward_pre_hooks
  )
  self._forward_pre_hooks[handle.id] = hook
  return handle

def register_buffer(self, name, tensor):
  # 不走 __setattr__，直接操作字典
  self._buffers[name] = tensor
  # buffer 会被 to() 移动
  # 会被 state_dict() 保存
  # 但不会被 parameters() 返回
  # → optimizer 不会更新它

class RemovableHandle:
  def __init__(self, hooks_dict):
    self.hooks_dict = hooks_dict
    self.id = id(self)  # 唯一 ID

  def remove(self):
    del self.hooks_dict[self.id]
使用场景
register_forward_hook — MoReS 用法：

def intervention_hook(module, input, output):
    h = output[0]
    h_new = h + R_T @ (W(h) - R @ h)
    return (h_new,) + output[1:]

handle = model.layers[0] \
  .register_forward_hook(intervention_hook)

→ model.layers[0]._forward_hooks = {
    140234567890: intervention_hook
  }

handle.remove()
→ _forward_hooks = {}  ← 清空了

register_buffer — 存不训练但要保存的东西：

self.register_buffer("mask", torch.ones(3,3))

→ self._buffers = {"mask": tensor(3,3)}

对比:
  nn.Parameter  → _parameters → 训练+保存+移动
  register_buffer → _buffers → 不训练+保存+移动
  普通赋值      → 无字典     → 不训练+不保存+不移动

一张图：哪些东西存在哪里，谁能遍历到它

self.xxx = yyy ← __setattr__ 自动分拣 │ ┌────────────┼────────────┐ ▼ ▼ ▼ _parameters _modules 普通属性 (Parameter) (Module) (其他一切) │ │ │ │ 各自也有 不在任何 │ _parameters 字典里 │ _modules ↓ │ _buffers parameters() ✗ │ │ state_dict() ✗ ▼ ▼ to() ✗ .parameters() 递归遍历 ✓ register_buffer .state_dict() 递归遍历 ✓ → _buffers .to(device) 递归遍历 ✓ │ .train/eval 递归遍历 ✓ state_dict() ✓ .__call__ → hooks to() ✓ parameters() ✗ ← 不训练
一句话：nn.Module = 几个 OrderedDict + 所有方法都递归遍历这些字典。赋值时 __setattr__ 自动分拣，调用时 __call__ 自动插 hook。不在字典里的东西，什么自动化功能都享受不到。
