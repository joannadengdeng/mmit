MoReS: Representation Steering

Bi et al., "LLaVA Steering: Visual Instruction Tuning through Representation Steering", ACL 2025

MoReS 在做什么？

一句话

不改模型权重，只在特定层的输出上"轻推"一下方向。

想象 hidden state 是一个在高维空间飞行的球：
• LoRA：改变发射台（修改权重矩阵 W）
• MoReS：不改发射台，在飞行途中轻推球的方向

这个"轻推"只在一个极小的子空间（rank=4）里发生，所以参数量比 LoRA 少 500 倍。

公式

h' = h + RT · (W·h + b - R·h)

R: [d, D]  正交投影矩阵
W: [d, D]  学习的"转向"矩阵
d = 4      极低维子空间
D = 1536   模型 hidden 维度

为什么叫 "Steering"（转向）？

W·h 是模型认为 h 应该去的方向
R·h 是 h 当前在子空间里的投影
W·h - R·h = 两者的差 = 需要修正的量
RT · (...) = 把修正量映射回原始高维空间

所以整个过程是：在低维空间里算出要修正多少，然后映射回去加上。

关键数字（Qwen2.5-VL-3B）

模型总参数: ~3B
LoRA 参数 (r=8): ~3.5M
MoReS 参数 (d=4, 9层): ~55K + projector
参数减少: ~60x fewer
投影到底在做什么？

日常类比

太阳照在你身上，地面上出现你的影子。

你是 3D 的（高维），影子是 2D 的（低维）。
影子保留了你的轮廓信息，但丢失了高度。

R·h 就是在做同样的事：
h 是 1536 维的"你"，
R·h 是 4 维的"影子"。

为什么 MoReS 需要投影？

直接在 1536 维上做修正 → 需要 1536×1536 的矩阵 → 236 万参数

先投影到 4 维，在 4 维上修正，再投影回来：
R: 4×1536 + W: 4×1536 = 1.2 万参数

省了 200 倍！代价是：修正只能发生在 R 定义的 4 维子空间里。

R·h vs RT·x 的关系

R·h: 高维 → 低维（投影/压缩）
RT·x: 低维 → 高维（反投影/展开）

如果 R 是正交的：
RT·R·h = h 在子空间里的分量
（不等于 h 本身！因为丢失了子空间外的信息）

就像影子没法完全还原成 3D 的你。
但 MoReS 不需要还原——它只需要在子空间里算出修正量，然后映射回去加上。

拖动左图的滑块试试

改变投影方向（R 的方向），看看"影子"如何变化。
不同的 R 会捕捉到 h 的不同方面。
正交约束：为什么 R 必须正交？

什么是正交矩阵？

满足 R·RT = I（单位矩阵）的矩阵。

几何意义：R 的行向量互相垂直，且长度为 1。
像一组完美的坐标轴——不重叠、不压缩。

正交为什么重要？

1. 投影不丢信息：
R·h 把 h 投影到低维空间，RT 再投影回来。
如果 R 正交，RT·R·h 只会保留 h 在子空间中的分量，不会引入畸变。

2. 修正量不泄漏：
δ = RT·(...)，如果 R 正交，δ 严格落在 R 定义的子空间内。
不会意外修改子空间外的维度。

3. 训练稳定：
正交约束防止 R 退化（比如所有行变成同一个方向）。

软约束 vs 硬约束

硬约束：每步训练后强制正交化（Gram-Schmidt）—— 计算贵
软约束（MoReS 用的）：加一个惩罚项到 loss
L_ortho = ||R·RT - I||F2

L_total = L_task + λ · L_ortho

# 代码：
def orthogonal_penalty(self):
    RRt = self.R @ self.R.t()
    I = torch.eye(self.rank)
    return (RRt - I).pow(2).sum()
当 R 完美正交时，RRT=I，惩罚=0。
偏离越多，惩罚越大。
Hook 挂在哪里？

MoReS vs LoRA 的挂载点不同

LoRA：挂在每个 nn.Linear 上（q/k/v/o/gate/up/down_proj）
每层 7 个挂载点

MoReS：挂在整个 Transformer Layer 的输出上
每层只有 1 个挂载点

而且 MoReS 不是每层都挂——只选前几层 + 后几层：
"f4+l5" = 前 4 层 + 后 5 层 = 9 个 intervention

为什么只选部分层？

论文发现：
• 前几层：处理视觉信息的编码
• 后几层：生成语言输出的关键
• 中间层：更通用的表示，改动收益小

选 f4+l5 是 accuracy-efficiency 的最佳平衡。

对应代码

# 位置解析
positions = _parse_positions("f4+l5", 36)
# → [0,1,2,3, 31,32,33,34,35]

# 在这些层上挂 hook
for idx in positions:
  iv = OrthogonalIntervention(
      hidden_dim=1536, rank=4)
  layer_list[idx].register_forward_hook(
      make_hook(iv))

还训练了什么？

除了 9 个 intervention 模块，MoReS 还解冻了
vision-language projector（视觉→语言的桥梁）。

LLM 的所有权重完全不动。
MoReS vs LoRA

本质区别

LoRA：改变 W 的行为（W+ΔW）
→ 改变线性变换本身
→ 影响 attention pattern / FFN 映射

MoReS：不改 W，改 hidden state h
→ 在层与层之间"拐弯"
→ W 的行为完全不变，只是输入被微调了

参数量对比

LoRA (r=8, 252 个 Linear):
  每个: 1536*8 + 8*1536 = 24,576
  总计: 252 * 24,576 = 6.19M

MoReS (d=4, 9 层):
  R: 4 * 1536 = 6,144
  W: 4 * 1536 + 4 = 6,148
  每层: 12,292
  总计: 9 * 12,292 = 110K

MoReS 约为 LoRA 的 1/56

什么时候用哪个？

用 LoRA：
• 需要大幅改变模型行为
• 任务和预训练差距大
• GPU 显存充足

用 MoReS：
• 微调幅度小（如领域适配）
• 极度省参数/省显存
• 想保留更多预训练知识
