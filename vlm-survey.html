L2T: Learning to Instruct NeurIPS 2025

Zhou et al. · arXiv 2503.22215 · GitHub

核心思想：标准 VIT 只对 response token 算 loss。L2T 额外对指令 token也算 loss（去除模板后），迫使模型更好地关注视觉特征。可以叠加在任何方法上，0 额外参数，开销 <1%。

Loss 公式

Standard VIT: L = -Σ log p(X_A,i | X_V, X_I, X_A,<i) ← 只监督 response L2T: L = -Σ log p(X_I,i | X_V, X_I,<i) ← Learn to Instruct -Σ log p(X_A,i | X_V, X_I, X_A,<i) ← Learn to Respond 其中去除两类模板： 1. System 模板: "USER:", "ASSISTANT:", system prompt ← 简单模式匹配 2. Task 模板: "Answer using a single word..." ← 高频句子统计后去除

LiLoRA: LoRA in LoRA arXiv 2508.06202

Che et al. · arXiv

核心思想：CKA 分析发现不同任务的 LoRA A 矩阵高度相似，B 矩阵差异大。于是共享 A 和 B₀，每个任务只学小的 (B̃, Ã) 低秩分解，加余弦正则化防遗忘。每任务参数减少 70%，遗忘减少 70%。

架构公式

Standard LoRA: ΔW = B · A (B ∈ R^{d×r}, A ∈ R^{r×k}) LiLoRA task i: ΔW_i = (α · B₀ + (1-α) · B̃_i · Ã_i) · A B₀ ∈ R^{d×r} ← 共享基础矩阵（所有任务更新） A ∈ R^{r×k} ← 共享矩阵（所有任务更新） B̃_i ∈ R^{d×r̃} ← 任务特定低秩因子 Ã_i ∈ R^{r̃×r} ← 任务特定低秩因子 (r̃ < r) α ∈ (0,1) ← 可学习融合系数, init = Sigmoid(N(0,1)) Regularization: L_reg = (1 - cos(B̃_t·Ã_t, B̃_{t-1}·Ã_{t-1})) · ||B₀^t - B₀^{t-1}||² Total Loss: L = L_task + λ · L_reg

MoReS: LLaVA Steering ACL 2025

Bi et al. · arXiv 2412.12359 · GitHub

核心思想：VLM 存在模态不平衡——文本表示主导输出生成。MoReS 在每层对视觉 token 施加正交低秩投影转向。LLM 完全冻结，只需 0.164M 参数（比 LoRA 少 1150×），幻觉指标优于 Full FT。

数学公式

MoReS(h) = W_up · φ(h) φ(h) = Linear(h) - W_down · h ← 残差转向信号 W_down ∈ R^{d×D} ← 降维投影（D → d） W_up ∈ R^{D×d} ← 升维投影（d → D） 约束: W_down · W_up^T = I_D ← 正交约束，保证可逆 最优 rank: d = 1 （rank 1 效果最好！） 稀疏转向: 只对 1% 的视觉 token 做转向

Prompt 和答案处理规范

VQA 类标准 Prompt："Answer the question using a single word or phrase."

答案标准化（VQAv2 通用）：lowercase → 去句号（保留小数点）→ 数字词转数字（"two"→"2"）→ 去冠词（a/an/the）→ 处理缩写

推荐超参数

LoRA（充足显存 ≥ 1×A100）：
r=64-128, alpha=2×r, dropout=0.05, target=all_linear
lr=2e-4, bs=1/GPU×8_acc, epochs=1, cosine+3%warmup
QLoRA（有限显存 ≤ 24GB）：
r=8-32, alpha=2×r, dropout=0.05, target=q_proj,v_proj
4bit=NF4, double_quant=True, compute=bf16
lr=2e-4, bs=1/GPU×8_acc, epochs=1-3, cosine+3%warmup

关键规则

规则说明
Alpha = 2 × Rank最常见设置（Microsoft 原始推荐）
1 Epoch 是标准小数据集 (<10K) 可用 2-3 epochs
Cosine + 3% Warmup几乎通用于所有 VLM
AdamW统一选择（8-bit 版本可省显存）
Vision LR = LLM LR / 5~10如果解冻 Vision Encoder
Weight Decay0.0（LLaVA）~ 0.05（InternVL）都行
BS=1/GPU + Grad AccumVLM 图像占显存，单卡 bs=1 常见
Gradient Checkpointing大模型必开
不要量化 ViTVision 部分保持 fp16/bf16
