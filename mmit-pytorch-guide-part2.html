Day 3 — 注意力机制

3.1 Scaled Dot-Product Attention — 注意力的核心

核心公式

Attention(Q, K, V) = softmax(Q · KT / √dk) · V

三个矩阵各有其语义：Q（Query）= "我在找什么？"，K（Key）= "我包含什么？"，V（Value）= "我输出什么？"。

直觉：对于每个 Query，计算它与所有 Key 的相似度，然后用这些相似度作为权重，对 Value 做加权求和。

为什么要除以 √dk？

当 dk 较大时，Q · KT 的值也会变大（方差约为 dk）。大数值传入 softmax 会导致梯度消失（softmax 输出接近 one-hot）。除以 √dk 使得方差回到 1，softmax 输出更平滑。

PyTorch 实现

import math
import torch.nn.functional as F

# Q, K, V: (batch, seq_len, d_k)
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)   # (B, seq, seq)
weights = F.softmax(scores, dim=-1)                   # 每行和为 1
output = weights @ V                                     # (B, seq, d_k)

F.softmax(x, dim=-1)对最后一维做 softmaxdim 指定归一化维度
mmit 关联：理解注意力是理解 LoRA 注入点的前提。LoRA 注入在 q_proj、k_proj、v_proj、o_proj 这四个线性层上，它们分别负责生成 Q、K、V 和投影输出。

3.2 Causal (Masked) Self-Attention — 因果注意力

概念

因果注意力和标准注意力公式相同，但增加了一个约束：每个位置只能关注自己和之前的位置，不能看到未来的 token。这是自回归生成（逐字预测）的基础。

实现方式：在 softmax 之前，将所有"未来位置"的分数设为 -∞，softmax 后这些位置的权重自然变为 0。

因果掩码可视化

pos 01 (可见)0 (-∞)0 (-∞)0 (-∞)
pos 11 (可见)1 (可见)0 (-∞)0 (-∞)
pos 21 (可见)1 (可见)1 (可见)0 (-∞)
pos 31 (可见)1 (可见)1 (可见)1 (可见)
第 i 行第 j 列：位置 i 能否关注位置 j？绿色=可以，红色=不可以
masked_fill 的效果
原始分数
[3.2, 1.8, 0.5, 0.3]
→

[3.2, -∞, -∞, -∞]
→

位置 0 只能关注自己，所以权重全部集中在位置 0

为什么需要因果掩码？

自回归生成时，模型逐个预测下一个 token。如果位置 i 能看到位置 i+1 的内容，那就是"作弊"——预测下一个词时已经知道答案了。因果掩码确保模型只用过去信息来预测未来。

PyTorch 实现

# 生成上三角掩码（True 表示需要屏蔽的位置）
L = seq_len
mask = torch.triu(torch.ones(L, L), diagonal=1).bool()  # (L, L)

# 在 softmax 之前应用掩码
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)
scores.masked_fill_(mask, float('-inf'))    # 未来位置设为 -inf
weights = F.softmax(scores, dim=-1)
output = weights @ V
API作用关键参数
torch.triu(x, diagonal=1)取上三角（不含对角线）diagonal=1 排除对角线
scores.masked_fill_(mask, val)在 mask=True 的位置填入 val原地操作，节省内存
mmit 关联：因果掩码（attention 层面）和 labels=-100（loss 层面）都是"屏蔽"的思想。因果掩码防止模型看到未来 token；labels=-100 让 loss 忽略 instruction 部分。mmit 的 L2T 方法就是改变 loss 层面的 masking 策略。两者层级不同但理念相通。

3.3 Multi-Head Attention — 多头注意力

概念

将 Q、K、V 各自拆分成 H 个"头"，每个头独立计算注意力，最后将所有头的结果拼接起来。

为什么要多头？单头注意力只能学到一种"关注模式"。多头注意力让每个头专注于不同的关系——比如一个头关注句法关系，另一个头关注语义相似度，又一个头关注位置关系。

LLaVA-1.5-7B 的具体参数

hidden_dim = 4096, num_heads = 32, head_dim = 128 (4096 / 32 = 128)

PyTorch 实现

class MultiHeadAttention(nn.Module):
    def __init__(self, hidden_dim=4096, num_heads=32):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = hidden_dim // num_heads   # 128
        self.q_proj = nn.Linear(hidden_dim, hidden_dim)
        self.k_proj = nn.Linear(hidden_dim, hidden_dim)
        self.v_proj = nn.Linear(hidden_dim, hidden_dim)
        self.o_proj = nn.Linear(hidden_dim, hidden_dim)

    def forward(self, x):
        B, S, _ = x.shape
        # 投影 + 拆分成多头
        Q = self.q_proj(x).view(B, S, self.num_heads, self.head_dim).transpose(1, 2)
        K = self.k_proj(x).view(B, S, self.num_heads, self.head_dim).transpose(1, 2)
        V = self.v_proj(x).view(B, S, self.num_heads, self.head_dim).transpose(1, 2)
        # (B, num_heads, S, head_dim)

        scores = Q @ K.transpose(-2, -1) / math.sqrt(self.head_dim)
        weights = F.softmax(scores, dim=-1)
        attn_out = weights @ V  # (B, num_heads, S, head_dim)

        # 合并多头
        attn_out = attn_out.transpose(1, 2).contiguous().view(B, S, -1)
        return self.o_proj(attn_out)
投影层形状作用LoRA 注入？
q_proj(4096, 4096)生成 Query是
k_proj(4096, 4096)生成 Key是
v_proj(4096, 4096)生成 Value是
o_proj(4096, 4096)输出投影是
mmit 关联：LoRA 的 target_modules 通常设为 ["q_proj", "k_proj", "v_proj", "o_proj"]。这四个投影矩阵是注意力机制中唯一的可学习参数（不含 MLP 部分）。LoRA 在这些层旁边注入低秩适配器 W = W0 + BA。

3.4 LayerNorm — 层归一化

概念

公式：y = (x - mean) / √(var + ε) × γ + β

对每个样本的特征维度进行归一化，使均值为 0、方差为 1，然后用可学习参数 γ（缩放）和 β（偏移）恢复表达能力。

为什么需要？训练过程中各层的激活值分布会漂移（internal covariate shift），导致训练不稳定。归一化让激活值始终在合理范围内。

归一化过程可视化

LayerNorm 处理单个向量 (特征维度=4)
输入 x
[2.0, 4.0, 6.0, 8.0]
mean=5.0, var=5.0
↓
(x - mean) / √(var + ε)
归一化后
[-1.34, -0.45, 0.45, 1.34]
mean≈0, std≈1
↓
× γ + β（可学习参数）
γ (4,)
β (4,)
→
输出 y
形状不变：(B, seq, 4096) → (B, seq, 4096)，均值和方差在最后一维(4096)上计算

PyTorch 实现

# 方式 1：nn.Module
ln = nn.LayerNorm(4096)          # gamma 和 beta 各 4096 个参数
output = ln(x)                    # x: (B, seq, 4096)

# 方式 2：函数式
output = F.layer_norm(x, [4096], weight=gamma, bias=beta)

# gamma 初始化为 1，beta 初始化为 0 → 初始时相当于仅归一化
API作用可学习参数数量
nn.LayerNorm(d)创建 LayerNorm 层2 × d（gamma + beta）
F.layer_norm(x, shape)函数式 LayerNorm需手动传入 weight/bias
mmit 关联：标准 Transformer 使用 LayerNorm，但 LLaMA 系列使用 RMSNorm（见 3.5）。归一化层通常不作为 LoRA 的注入目标，但在模型冻结 / 解冻策略中需要注意这些层的梯度状态。

3.5 RMSNorm — LLaMA 用的归一化

概念

公式：y = x / RMS(x) × γ，其中 RMS(x) = √(mean(x²) + ε)

与 LayerNorm 的区别：不减均值，不用 β，只用 γ。计算更快，效果相当。

对比表

特性LayerNormRMSNorm
减均值是 (x - mean)否
除以什么√(var + ε)√(mean(x²) + ε)
可学习参数γ + β (2d 个)仅 γ (d 个)
速度较慢（需计算 mean 和 var）较快（只算 mean(x²)）
使用者BERT, GPT-2, ViTLLaMA, Mistral, Qwen

LayerNorm

(x-5)/√5 → [-1.34, -0.45, 0.45, 1.34]

RMSNorm
mean(x²)=30, RMS=√30≈5.48
x/5.48 → [0.37, 0.73, 1.10, 1.46]
×γ

PyTorch 实现

# PyTorch 没有内置 RMSNorm，需自定义或从 transformers 导入
class RMSNorm(nn.Module):
    def __init__(self, dim, eps=1e-6):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(dim))  # gamma

    def forward(self, x):
        rms = torch.sqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)
        return x / rms * self.weight

# 或者直接用 HuggingFace 的实现
from transformers.models.llama.modeling_llama import LlamaRMSNorm
mmit 关联：LLaVA 使用的 LLaMA 骨架中，每个 Transformer Block 有两个 RMSNorm：一个在 Self-Attention 之前（input_layernorm），一个在 MLP 之前（post_attention_layernorm）。理解这一点有助于阅读 mmit 中 MoReS hook 的插入位置。

Day 4 — 模型结构

4.1 Transformer Block — 一层 Transformer

概念

Transformer Block 是 Transformer 模型的重复单元。LLaVA-1.5-7B 将 32 个这样的 Block 依次堆叠。

LLaMA 采用 Pre-Norm 结构（先归一化再计算），而非原始 Transformer 的 Post-Norm。

完整结构可视化

LLaMA-style Transformer Block（Pre-Norm + 残差连接）
输入 x ─────────────────────────────────────┐ (残差连接) │ │ ▼ │ ┌──────────────┐ │ │ RMSNorm │ input_layernorm │ └──────┬───────┘ │ ▼ │ ┌──────────────────────────────────────┐ │ │ Multi-Head Self-Attention │ │ │ ┌────────┬────────┬────────┐ │ │ │ │ q_proj │ k_proj │ v_proj │ ← LoRA │ │ │ └────┬───┴────┬───┴────┬───┘ │ │ │ ▼ ▼ ▼ │ │ │ Attention + Causal Mask │ │ │ │ │ │ │ ┌────▼───┐ │ │ │ │ o_proj │ ← LoRA │ │ │ └────┬───┘ │ │ └───────┼──────────────────────────────┘ │ ▼ │ (+) ◄──────────────────────────────────┘ 残差相加 │ ├──────────────────────────────────┐ (残差连接) ▼ │ ┌──────────────┐ │ │ RMSNorm │ post_attention_layernorm │ └──────┬───────┘ │ ▼ │ ┌──────────────────────────────────────┐ │ │ SwiGLU MLP │ │ │ ┌───────────┐ ┌──────────┐ │ │ │ │ gate_proj │ │ up_proj │ ← LoRA │ │ │ └─────┬─────┘ └────┬─────┘ │ │ │ SiLU ▼ │ │ │ │ ├──── × ──────┘ │ │ │ ▼ │ │ │ ┌───────────┐ │ │ │ │ down_proj │ ← LoRA │ │ │ └─────┬─────┘ │ │ └────────┼─────────────────────────────┘ │ ▼ │ (+) ◄────────────────────────────────┘ 残差相加 │ ▼ 输出 ─── → MoReS 在这里 hook!

残差连接的作用

output = x + f(x)。允许梯度直接通过"加法"回传，解决深层网络的梯度消失问题。即使 f(x) 的梯度很小，x 这条路径仍然畅通。

PyTorch 实现

class TransformerBlock(nn.Module):
    def __init__(self, hidden_dim, num_heads):
        super().__init__()
        self.input_layernorm = RMSNorm(hidden_dim)
        self.self_attn = MultiHeadAttention(hidden_dim, num_heads)
        self.post_attention_layernorm = RMSNorm(hidden_dim)
        self.mlp = SwiGLU_MLP(hidden_dim)

    def forward(self, x):
        # Pre-Norm + Self-Attention + 残差
        h = x + self.self_attn(self.input_layernorm(x))
        # Pre-Norm + MLP + 残差
        out = h + self.mlp(self.post_attention_layernorm(h))
        return out
LoRA 注入目标所属子模块参数形状
q_proj, k_proj, v_proj, o_projSelf-Attention(4096, 4096) 各一个
gate_proj, up_projMLP(11008, 4096) 各一个
down_projMLP(4096, 11008)
mmit 关联：MoReS 在每个 Block 的输出端注册 hook（register_forward_hook），捕获该层的输出并施加方向控制（steering）。LoRA 则注入到 Block 内部的 Linear 层。理解 Block 的结构就能理解这两种方法的注入位置差异。

4.2 SwiGLU MLP — LLaMA 的 MLP

概念

公式：MLP(x) = down_proj( SiLU(gate_proj(x)) ⊙ up_proj(x) )

SiLU（Swish）激活函数：SiLU(x) = x × sigmoid(x)，是 ReLU 的平滑版本，允许小的负值通过。

双分支结构可视化

SwiGLU MLP 的两分支 + 门控结构
x (B, seq, 4096)
↓
gate_proj
Linear(4096, 11008)
↓
SiLU 激活
(B, seq, 11008)
↓
up_proj
Linear(4096, 11008)
↓
直接输出
(B, seq, 11008)
↓
逐元素相乘 (⊙)
(B, seq, 11008)
↓
down_proj   Linear(11008, 4096)
↓
输出 (B, seq, 4096)

为什么用 SwiGLU？

标准 FFN 只有一次线性变换 + ReLU + 一次线性变换。SwiGLU 用门控机制（gate 分支控制信息流通量）代替简单的 ReLU，实验表明效果更好。PaLM、LLaMA、Mistral 等现代模型都采用了这个设计。

PyTorch 实现

class SwiGLU_MLP(nn.Module):
    def __init__(self, hidden_dim=4096, intermediate_dim=11008):
        super().__init__()
        self.gate_proj = nn.Linear(hidden_dim, intermediate_dim, bias=False)
        self.up_proj   = nn.Linear(hidden_dim, intermediate_dim, bias=False)
        self.down_proj = nn.Linear(intermediate_dim, hidden_dim, bias=False)

    def forward(self, x):
        gate = F.silu(self.gate_proj(x))   # SiLU = x * sigmoid(x)
        up   = self.up_proj(x)
        return self.down_proj(gate * up)    # 逐元素相乘后降维
层参数形状参数量说明
gate_proj(11008, 4096)~45M门控分支
up_proj(11008, 4096)~45M值分支
down_proj(4096, 11008)~45M降维投影
mmit 关联：gate_proj、up_proj、down_proj 都是 LoRA 可注入的目标。mmit 的 LoRA 配置中 target_modules 通常同时包含注意力层的 4 个投影和 MLP 的 3 个投影，共 7 个线性层 × 32 个 Block = 224 个注入点。

4.3 RoPE — 旋转位置编码

概念

传统方法：给 token embedding 加上位置向量。RoPE 的做法不同——旋转 Q 和 K 向量，使得它们的点积自然编码位置信息。

核心思想：将向量的相邻两个维度看作 2D 平面上的一个点，根据位置 m 旋转角度 mθ。旋转后，Qm · Kn 的结果只取决于相对位置 (m-n)，而非绝对位置。

旋转可视化

2D 旋转：相邻维度配对，按位置旋转
位置 0: 旋转 0°
q = [q0, q1]
→ [q0, q1]（不动）
位置 5: 旋转 5θ
q = [q0, q1]
→ [q0cos5θ - q1sin5θ,
    q0sin5θ + q1cos5θ]
每对维度 (2i, 2i+1) 使用不同的基础频率 θi = 10000-2i/d
维度 0-1: θ=1.0 → 高频旋转（编码近距离关系）
维度 2-3: θ=0.01 → 中频旋转
维度 126-127: θ≈0.0001 → 低频旋转（编码远距离关系）

RoPE 的优势

特性绝对位置编码RoPE
编码方式加法：x + PE旋转：对 Q, K 做旋转
位置信息类型绝对位置相对位置（自然从旋转差中得到）
外推能力差（超出训练长度时性能下降）好（旋转角度可自然外推）
使用者BERT, GPT-2LLaMA, Mistral, Qwen, PaLM

PyTorch 实现（简化版）

def apply_rotary_emb(q, k, positions, dim):
    # 计算旋转角度: theta_i = 10000^(-2i/dim)
    freqs = 1.0 / (10000 ** (torch.arange(0, dim, 2).float() / dim))
    # positions: (seq_len,) → angles: (seq_len, dim//2)
    angles = positions[:, None] * freqs[None, :]
    cos_a, sin_a = angles.cos(), angles.sin()

    # 将 q 的相邻维度配对
    q_even, q_odd = q[..., ::2], q[..., 1::2]
    q_rot = torch.stack([
        q_even * cos_a - q_odd * sin_a,
        q_even * sin_a + q_odd * cos_a
    ], dim=-1).flatten(-2)
    # 对 k 做同样操作...
    return q_rot, k_rot
mmit 关联：LLaVA 使用 RoPE，这意味着视觉 token 和文本 token 拼接后，视觉 token 会被赋予前面的位置编号。理解 RoPE 有助于调试注意力模式和理解不同长度输入的行为。

用 Conv2d 实现 Patch Embedding

巧妙地利用 stride = patch_size 的卷积，一步完成切割和投影：

# kernel_size = stride = patch_size, 所以卷积核不重叠
patch_embed = nn.Conv2d(
    in_channels=3,           # RGB
    out_channels=1024,       # embedding 维度
    kernel_size=14,          # patch 大小
    stride=14                # 步长 = patch 大小（不重叠）
)
# 输入: (B, 3, 336, 336)
# 输出: (B, 1024, 24, 24) → reshape → (B, 576, 1024)
# CLIP ViT-L/14@336: 24×24 = 576 个视觉 token

tokens = patch_embed(image)         # (B, 1024, 24, 24)

4.5 Cross-Attention — 跨模态注意力

概念

在自注意力（Self-Attention）中，Q、K、V 都来自同一个序列。而在交叉注意力中，Q 来自一个序列，K 和 V 来自另一个序列，让模型能"看到"来自不同模态的信息。

Self-Attention vs Cross-Attention

两种注意力的对比
Self-Attention（自注意力）
同一序列 x
Q = Wq · x
K = Wk · x
V = Wv · x
Q, K, V 都来自 x
Cross-Attention（交叉注意力）
文本 token
视觉 token
Q = Wq · 文本
K = Wk · 视觉
V = Wv · 视觉
Q 来自文本，K/V 来自视觉
Cross-Attention 连接文本与视觉
文本 token: [T₀] [T₁] [T₂] [T₃] ← 生成 Q ╲ │ ╱ ╲ 注意力权重 ╱ │ ╲ ╱ 视觉 token: [V₀] [V₁] [V₂] [V₃] ← 生成 K, V 每个文本 token 的输出 = 视觉 token 的加权求和 → 文本 token "看到了" 图像内容

PyTorch 实现

class CrossAttention(nn.Module):
    def __init__(self, dim):
        super().__init__()
        self.q_proj = nn.Linear(dim, dim)
        self.k_proj = nn.Linear(dim, dim)
        self.v_proj = nn.Linear(dim, dim)

    def forward(self, text_tokens, visual_tokens):
        Q = self.q_proj(text_tokens)      # (B, S_text, dim)
        K = self.k_proj(visual_tokens)    # (B, S_visual, dim)
        V = self.v_proj(visual_tokens)    # (B, S_visual, dim)
        # attention: (B, S_text, S_visual)
        scores = Q @ K.transpose(-2, -1) / math.sqrt(Q.size(-1))
        weights = F.softmax(scores, dim=-1)
        return weights @ V  # (B, S_text, dim)
mmit 关联：有些 VLM 架构（如 Flamingo）使用 Cross-Attention 做视觉-语言融合。但 LLaVA 不用 Cross-Attention——它直接将视觉 token 和文本 token 拼接成一个序列，然后用普通的 Self-Attention（因果掩码）处理所有 token。这种设计更简单，但意味着视觉 token 会占用 Self-Attention 的序列长度。

class LLaVA(nn.Module):
    def forward(self, image, input_ids, labels):
        # Step 1: 视觉编码
        visual_features = self.clip_vit(image)       # (B, 576, 1024)

        # Step 2: 投影到 LLM 空间
        visual_tokens = self.projector(visual_features) # (B, 576, 4096)

        # Step 3: 文本编码
        text_tokens = self.embed_tokens(input_ids)   # (B, seq, 4096)

        # Step 4: 拼接
        hidden = torch.cat([visual_tokens, text_tokens], dim=1)

        # Step 5: 通过 32 层 Transformer Block
        for block in self.layers:
            hidden = block(hidden)                   # 每层都有 LoRA / MoReS

        # Step 6: 输出 logits 和计算 loss
        logits = self.lm_head(self.norm(hidden))     # (B, total_seq, 32000)
        loss = F.cross_entropy(
            logits[:, :-1].reshape(-1, 32000),
            labels[:, 1:].reshape(-1),
            ignore_index=-100                       # 忽略 instruction token
        )
        return loss
