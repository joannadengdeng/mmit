1. MoLE — 混合 LoRA 专家 (Chen et al., 2024, arXiv 2401.16160)

1.1 核心问题和方案

问题：LLaVA-1.5 用 665K 混合数据训练（VQA + OCR + Science + Caption），单个 LoRA adapter 被不同类型的数据往不同方向拉扯，产生参数干扰。

方案：每个 MLP 层放 4 个 LoRA "专家" + 1 个 router。每个 token 由 router 选 1 个最匹配的专家执行。注意力层仍用标准单 LoRA。

1.2 LoRAExpert 源码

# mole.py:37-50 LoRAExpert — 一个 LoRA 专家
class LoRAExpert(nn.Module):
    def __init__(self, in_features, out_features, rank, dropout=0.0):
        super().__init__()
        self.lora_A = nn.Linear(in_features, rank, bias=False)   # 降维: 4096 → 8
        self.lora_B = nn.Linear(rank, out_features, bias=False)  # 升维: 8 → 11008
        self.dropout = nn.Dropout(dropout) if dropout > 0 else nn.Identity()
        nn.init.kaiming_uniform_(self.lora_A.weight, a=5**0.5)   # Kaiming init
        nn.init.zeros_(self.lora_B.weight)                         # B 初始化为 0 → 初始 delta=0

    def forward(self, x):
        return self.lora_B(self.lora_A(self.dropout(x)))
        # x: (N, 4096) → A: (N, 8) → B: (N, 11008)

# 每个 expert 参数: 4096×8 + 8×11008 = 32,768 + 88,064 = 120,832

1.3 MoLELayer 源码 — Router + 多专家 dispatch

# mole.py:53-122 MoLELayer — 包装一个 frozen Linear + N 个专家 + router
class MoLELayer(nn.Module):
    def __init__(self, base_linear, num_experts, rank, alpha, dropout):
        self.base_linear = base_linear                  # frozen 的原始 Linear
        self.num_experts = num_experts                  # 4
        self.scaling = alpha / rank                     # 16/8 = 2.0

        # 冻结原始权重
        for p in base_linear.parameters():
            p.requires_grad = False

        # 4 个 LoRA 专家
        self.experts = nn.ModuleList([
            LoRAExpert(in_f, out_f, rank, dropout) for _ in range(num_experts)])

        # Router: 一个简单的 Linear → softmax
        self.router = nn.Linear(in_f, num_experts, bias=False)  # (4096 → 4)

    def forward(self, x):
        # x: (batch, seq, 4096)

        # 1. 原始 frozen 输出
        base_out = self.base_linear(x)          # (batch, seq, 11008)

        # 2. Router 选专家
        router_logits = self.router(x.detach()) # (batch, seq, 4) ← detach 防止梯度回传到 router 输入
        routing_probs = F.softmax(router_logits, dim=-1)  # (batch, seq, 4)
        expert_idx = routing_probs.argmax(dim=-1)          # (batch, seq) ← top-1

        # 3. Dispatch: 每个 expert 处理分配给它的 token
        x_flat = x.reshape(-1, x.shape[-1])   # (620, 4096) ← flatten batch×seq
        idx_flat = expert_idx.reshape(-1)       # (620,)
        delta = torch.zeros(x_flat.shape[0], base_out.shape[-1], ...)  # (620, 11008)

        for i, expert in enumerate(self.experts):
            mask = (idx_flat == i)              # 哪些 token 分给 expert i
            if mask.any():
                delta[mask] = expert(x_flat[mask])  # 只跑选中的 token

        # 4. 加到 frozen 输出上
        return base_out + delta.reshape(*orig_shape, -1) * self.scaling
        # (batch, seq, 11008) = frozen + expert_delta × 2.0

输入 h: (1, 620, 4096)
  ├─ 位置 0~31: 系统prompt + USER (文本)
  ├─ 位置 32~607: 576 个 visual tokens (公交车图)
  └─ 位置 608~619: 问题 + answer

Step 1: base_out = gate_proj(h)  → (1, 620, 11008)  [frozen, 不训练]

Step 2: Router
  router_logits = router(h.detach())  → (1, 620, 4)
  routing_probs = softmax(router_logits)
  假设:
    位置 32 (公交车主体 visual token):  probs = [0.12, 0.65, 0.15, 0.08] → 选 Expert 1
    位置 100 (天空 visual token):       probs = [0.55, 0.20, 0.10, 0.15] → 选 Expert 0
    位置 610 ("colors" 文本 token):    probs = [0.08, 0.10, 0.72, 0.10] → 选 Expert 2

Step 3: Dispatch
  Expert 0 处理: ~200 个 token (背景 visual + 系统 prompt)
  Expert 1 处理: ~300 个 token (主体 visual, 公交车相关)
  Expert 2 处理: ~100 个 token (文本 token, 问题相关)
  Expert 3 处理: ~20 个 token  (其他)

Step 4: base_out + delta × 2.0
  每个位置只有 1 个专家的 delta, 不是全部 4 个
  → 训练后不同专家学到不同的"修正方向"

1.6 Load-balancing loss 源码

# mole.py:279-303 _load_balance_loss() 完整代码
def _load_balance_loss(self):
    """L_balance = N × Σᵢ (fᵢ × Pᵢ)"""
    total_loss = 0.0
    count = 0
    for layer in self._mole_layers:       # 96 个 MoLELayer
        if layer._routing_probs is None:
            continue
        probs = layer._routing_probs.reshape(-1, layer.num_experts)  # (620, 4)
        mask = layer._expert_mask.reshape(-1, layer.num_experts)    # (620, 4) one-hot

        f = mask.mean(dim=0)   # (4,) 每个 expert 分到的 token 比例
        # 例: f = [0.32, 0.48, 0.16, 0.04] ← Expert 1 分到最多

        P = probs.mean(dim=0)  # (4,) 每个 expert 的平均 routing 概率
        # 例: P = [0.30, 0.45, 0.15, 0.10]

        total_loss = total_loss + (f * P).sum() * layer.num_experts
        # = (0.32×0.30 + 0.48×0.45 + 0.16×0.15 + 0.04×0.10) × 4
        # = (0.096 + 0.216 + 0.024 + 0.004) × 4 = 1.36
        # 如果完美均匀 (f=P=[0.25,0.25,0.25,0.25]):
        #   = 4 × (0.0625×4) = 1.0 ← 最小值
        count += 1
    return total_loss / count

# 最终: total_loss = CE_loss + 0.01 × L_balance

2. RandLoRA — Full-Rank 更新 (Kalajdzievski, ICLR 2025, arXiv 2502.00987)

2.1 核心问题

标准 LoRA: ΔW = B·A, rank(ΔW) = r。如果 r=8, d=4096, ΔW 只能在 4096 维空间的 8 维子空间里修正。很多任务需要更丰富的修正方向。

2.2 数学对比

标准 LoRA

ΔW = B · A

A: (d_in, r)  ← 训练
B: (r, d_out) ← 训练
scaling = alpha / r

rank(ΔW) = r
如果 r=8: 只能修正 8 个方向

RandLoRA (rsLoRA)

ΔW = B · A    (矩阵相同!)

A: (d_in, r)  ← 训练
B: (r, d_out) ← 训练
scaling = alpha / √r   ← 唯一区别!

rank(ΔW) = r (单看一样)
但: scaling 变了 → 高 rank 时更稳定
→ 可以安全使用更大的 r
→ 实际表达能力更强

2.4 用公交车例子看 scaling 的差异

📌 同一个 token 在 LoRA vs RandLoRA 下的 delta 差异

假设 q_proj 层, 某个 visual token (公交车主体), rank=8, alpha=16:

A·x = [0.5, -0.3, 0.8, 0.1, -0.6, 0.4, -0.2, 0.7]  (rank-8 投影, 两者相同)
B·(A·x) = [0.012, -0.008, 0.015, ..., 0.003]          (4096 维 delta, 两者相同)

标准 LoRA:  delta = B·A·x × (alpha / r) = B·A·x × (16/8) = B·A·x × 2.0
RandLoRA:   delta = B·A·x × (alpha / √r) = B·A·x × (16/√8) = B·A·x × 5.66

RandLoRA 的 delta 是 LoRA 的 2.83× 大!
→ 高 rank 时不会因为 scaling 太小导致 delta 消失
→ 训练更容易收敛, 尤其是 r=64 这种大 rank

实际效果:
  LoRA  r=64:  accuracy = 82.7%, scaling = 16/64 = 0.25 (很小!)
  rsLoRA r=64: accuracy = 85.7%, scaling = 16/√64 = 2.0  (合理)
  ↑ 同样参数量, +3.0 个点
一句话：RandLoRA 不改模型结构、不改参数量、不改训练流程。只改了 scaling factor 从 alpha/r 到 alpha/√r。用户只需在 YAML 里把 training_method: lora 改成 training_method: randlora。

3.2 LoReFTIntervention 完整源码

# reft.py:32-83 LoReFTIntervention 完整代码
class LoReFTIntervention(nn.Module):
    """h' = h + R^T @ (W @ (R @ h) + b - R @ h)"""

    def __init__(self, hidden_dim, rank=4, dropout=0.0):
        super().__init__()
        self.hidden_dim = hidden_dim   # 4096
        self.rank = rank               # 4

        # R: 投影矩阵 [rank, hidden_dim] — 正交初始化
        self.R = nn.Parameter(torch.empty(rank, hidden_dim))
        nn.init.orthogonal_(self.R)    # shape (4, 4096)

        # W: 子空间内的线性变换 [rank, rank]
        self.W = nn.Linear(rank, rank, bias=True)
        nn.init.eye_(self.W.weight)    # 初始化为单位矩阵 → 初始 W(Rh) = Rh → delta = 0
        nn.init.zeros_(self.W.bias)

        self.dropout = nn.Dropout(dropout) if dropout > 0 else nn.Identity()

    def orthogonal_penalty(self):
        """和 MoReS 一样的正交约束: ||R·R^T - I||²"""
        RRt = self.R @ self.R.t()                    # (4, 4)
        I = torch.eye(self.rank, device=self.R.device)
        return (RRt - I).pow(2).sum()

    def forward(self, h):
        # h: (batch, seq, 4096)
        Rh = F.linear(h, self.R)          # (B, seq, 4)    投影到子空间
        Wh = self.W(Rh)                   # (B, seq, 4)    子空间内变换
        delta_low = self.dropout(Wh - Rh) # (B, seq, 4)    差值
        delta = F.linear(delta_low, self.R.t())  # (B, seq, 4096) 投回原空间
        return h + delta

# 每个 intervention 的参数:
#   R: 4 × 4096 = 16,384
#   W.weight: 4 × 4 = 16
#   W.bias: 4
#   total = 16,404 / intervention

3.3 用公交车例子走一遍 ReFT forward (Layer 0)

📌 Layer 0 的 LoReFTIntervention 数据流

输入: h = layer_0_output, shape (1, 620, 4096)

Step 1: Rh = F.linear(h, R)    # R: (4, 4096)
  (1, 620, 4096) × (4096, 4)^T → (1, 620, 4)
  每个位置从 4096 维投影到 4 维
  例: h[0, 32] (第一个 visual token, 4096 维)
      → Rh[0, 32] = [-0.82, 0.45, 0.13, -0.67]  (4 维)

Step 2: Wh = self.W(Rh)        # W: Linear(4, 4)
  (1, 620, 4) → (1, 620, 4)
  训练初期 W ≈ I → Wh ≈ Rh → delta ≈ 0
  训练后: W 学到非单位变换
  例: Wh[0, 32] = [-0.90, 0.52, 0.08, -0.71]

Step 3: delta_low = Wh - Rh
  (1, 620, 4)
  例: delta_low[0, 32] = [-0.08, 0.07, -0.05, -0.04]

Step 4: delta = F.linear(delta_low, R.t())
  (1, 620, 4) × (4, 4096) → (1, 620, 4096)
  把 4 维差值沿 R 的 4 个方向展开回 4096 维
  例: delta[0, 32] = [0.003, -0.001, 0.005, ..., -0.002]  (4096 维, 很小)

Step 5: h' = h + delta
  (1, 620, 4096) + (1, 620, 4096) → (1, 620, 4096)
  所有 620 个位置都被修正 (不像 MoReS 只改 6 个)

3.5 ReFT 的 hook 注册 + 训练时 loss

# reft.py:181-211 注册 hook — 和 MoReS 几乎一样
for idx, iv in zip(target_indices, ivs_per_layer):
    def make_hook(intervention):
        def hook(module, input, output):
            h = output[0]                   # (1, 620, 4096)
            h = intervention(h)              # h' = h + R^T(W(Rh) - Rh)
            return (h,) + output[1:]
        return hook
    layer_list[idx].register_forward_hook(make_hook(iv))

# reft.py:247-261 compute_loss — CE + 正交约束
def compute_loss(self, model, batch, outputs):
    text_loss, metrics = _ce_loss.compute(model, batch, outputs)

    # 正交约束 (和 MoReS 完全一样)
    ortho_loss = 0.0
    for iv in self._interventions:         # 32 个
        ortho_loss = ortho_loss + iv.orthogonal_penalty()

    total = text_loss + 0.01 * ortho_loss
    return total, metrics
