模型 (VLM)

model_id = "llava-hf/llava-1.5-7b-hf"

架构: LLaVA-1.5
  ├─ 视觉编码器: CLIP ViT-L/14 (336×336)
  │    输出: 576 个 patch tokens (24×24)
  ├─ 投影层: 2 层 MLP
  │    把视觉特征映射到 LLM 空间
  └─ LLM: Vicuna-7B (LLaMA-2 base)
       hidden_dim = 4096
       num_layers = 32
       vocab_size = 32064

输入数据

image: "coco/train2017/000000033471.jpg"
       (一张公交车的图片)

question: "What are the colors
           of the buses?"

answer:   "The buses are white
           and red."
为什么用 LLaVA-1.5? MoLE 论文（LLaVA-MoLE）就是在 LLaVA 上做的。 LLaVA-1.5 的 MLP 结构清晰（gate_proj + up_proj + down_proj）， 方便展示 MoLE 怎么在 MLP 里注入多专家。每层 MLP 有 3 个 Linear， 每个 Linear 被替换成 MoLELayer（4 个 LoRA 专家 + 1 个 router）。

📌 关键概念 MoLE 在 LLM 内部改了什么？
MoLE 不用 hook——它直接替换 MLP 层的 Linear。每个 gate/up/down_proj 从单个 Linear 变成 MoLELayer（frozen Linear + 4 个 LoRA expert + router）
# MoLE 和 MoReS/LoRA 的关键区别: 改的位置不同 # # LoRA: 改 attention 的 q/k/v/o_proj (权重 += B·A) # MoReS: 改 layer 输出的 hidden state (h += delta, 用 hook) # MoLE: 改 MLP 的 gate/up/down_proj (直接替换 Linear 为 MoLELayer) # # LlamaDecoderLayer 内部 (Vicuna-7B, 每层相同): # ┌──────────────────────────────────────────┐ # │ RMSNorm → Self-Attention │ # │ q_proj: Linear(4096, 4096) ← 标准 LoRA │ # │ k_proj: Linear(4096, 4096) ← 标准 LoRA │ # │ v_proj: Linear(4096, 4096) ← 标准 LoRA │ # │ o_proj: Linear(4096, 4096) ← 标准 LoRA │ # │ + residual │ # │ RMSNorm → MLP (SwiGLU) │ # │ gate_proj: Linear(4096, 11008) ← ★ MoLELayer (4 experts + router) # │ up_proj: Linear(4096, 11008) ← ★ MoLELayer (4 experts + router) # │ down_proj: Linear(11008, 4096) ← ★ MoLELayer (4 experts + router) # │ + residual │ # └──────────────────────────────────────────┘ # # 32 层 × (3 MoLELayer + 4 Attn LoRA) = 96 MoLE + 128 LoRA # 总参数: ~56M (MoLE ~48M + Attn LoRA ~8M)
为什么 MLP 用 MoE 而 Attention 不用？MoLE 论文的 insight：数据冲突主要发生在 MLP 层（负责"记忆"知识的地方），Attention 层主要做"关注"不太受数据类型影响。所以 MLP 需要多专家分流，Attention 用单 LoRA 就够了。

# stage_runner.py 真实代码 def _cosine_schedule(optimizer, num_warmup, num_total): def lr_lambda(step): if step < num_warmup: return step / max(1, num_warmup) progress = (step - num_warmup) / max(1, num_total - num_warmup) return 0.5 * (1.0 + math.cos(math.pi * progress)) return LambdaLR(optimizer, lr_lambda) def _to_device(batch, device): out = {} for k, v in batch.items(): if isinstance(v, torch.Tensor): out[k] = v.to(device) elif isinstance(v, list) and v and isinstance(v[0], torch.Tensor): out[k] = [t.to(device) for t in v] else: out[k] = v return out

EMNLP 2023) 专门测幻觉: # 给模型看图片，问 "Is there a car?" → 模型答 yes/no # 如果图里没有车但模型说 yes → 幻觉! # # 判断逻辑: normalize(回答).startswith("yes") # "Yes, there is a car" → normalize → "yes there is car" → .startswith("yes") → True # "No" → normalize → "no" → .startswith("yes") → False tp = fp = tn = fn = 0 for pred in predictions: pred_yes = normalize_answer(pred["prediction"]).startswith("yes") gt_yes = normalize_answer(gt_map[pred["id"]]).startswith("yes") if pred_yes and gt_yes: tp += 1 # 真阳: 图里有车，模型说有 elif pred_yes: fp += 1 # 假阳: 图里没车，模型说有 ← 幻觉! elif not gt_yes: tn += 1 # 真阴: 图里没车，模型说没 else: fn += 1 # 假阴: 图里有车，模型说没 accuracy = (tp+tn) / (tp+fp+tn+fn) # 整体准确率 precision = tp / (tp+fp) # 模型说 yes 时，有多少是对的 recall = tp / (tp+fn) # 真的有的东西，模型找到了多少 f1 = 2*precision*recall/(precision+recall) # P 和 R 的调和平均 yes_rate = (tp+fp) / (tp+fp+tn+fn) # 模型说 yes 的比例 # ── POPE score 计算示例 (4 个样本) ── # # 样本1: 图里有车, gt="yes", 模型说"Yes" → tp (正确) # 样本2: 图里没车, gt="no", 模型说"Yes, there is" → fp (幻觉!) # 样本3: 图里没猫, gt="no", 模型说"No" → tn (正确) # 样本4: 图里有狗, gt="yes", 模型说"No" → fn (漏了) # # tp=1, fp=1, tn=1, fn=1 # accuracy = (1+1)/(1+1+1+1) = 0.5 = 50% # precision = 1/(1+1) = 0.5 = 50% # recall = 1/(1+1) = 0.5 = 50% # f1 = 2*0.5*0.5/(0.5+0.5) = 0.5 = 50% # yes_rate = (1+1)/4 = 0.5 = 50% # # yes_rate 很重要: 如果模型对所有问题都说 yes, # yes_rate=100%, accuracy~50% (因为大约一半 gt 是 no) # → 说明模型在瞎猜/有严重幻觉 # 理想的 yes_rate 应该接近 50% (和数据集的 yes/no 比例一致)

VQA 有 10 个标注者各自给答案——如果 ≥3 人同意你的回答，算满分
# ================================================================ # vqa.py:83-94 vqa_accuracy() — VQA v2 官方评分公式 # ================================================================ # VQA 数据集每个问题有 10 个标注者独立回答 # 比如 "What color?" → 10 人中 8 人说 "blue"，2 人说 "sky blue" # ground_truths = ["blue","blue","blue","blue","blue","blue","blue","blue", # "sky blue","sky blue"] def vqa_accuracy(prediction, ground_truths): norm_pred = normalize_answer(prediction) # "blue" # 数 10 个标注者中有几个同意这个答案 count = sum(1 for gt in ground_truths if normalize_answer(gt) == norm_pred) # count = 8 (8个人说了 "blue") return min(count / 3.0, 1.0) # = min(8/3, 1.0) = min(2.67, 1.0) = 1.0 ✓ 满分 # # 为什么除以 3？VQA v2 论文的规定: # 0 人同意 → 0/3 = 0.0 # 1 人同意 → 1/3 = 0.33 # 2 人同意 → 2/3 = 0.67 # 3+人同意 → min(k/3, 1.0) = 1.0 满分 # 含义: 至少 3 个标注者同意你的答案，就算完全正确 # aggregate_vqa_accuracy(): 所有样本取平均 # accuracy = mean([1.0, 0.33, 1.0, 0.0, ...]) = 0.583 # → 返回 {"accuracy": 58.3}

# vqa.py normalize_answer() 真实代码 def normalize_answer(answer): answer = answer.lower() # ② 缩写展开 (100+ 条) words = answer.split() words = [_CONTRACTIONS.get(w, w) for w in words] answer = " ".join(words) # ③ 标点移除 answer = "".join(ch for ch in answer if ch not in _PUNCT) # ④ 冠词移除 {"a", "an", "the"} words = [w for w in answer.split() if w not in _ARTICLES] answer = " ".join(words) # ⑤ 空格规范 return re.sub(r"\s+", " ", answer).strip()
5种 METRIC_REGISTRY — 5 种评分方式
vqa_accuracy (VQA Soft) min(#agree / 3, 1.0) 需要多人标注 (≥3人) exact_match normalize(pred) == any(normalize(gt)) → 1.0 / 0.0 token_f1 split tokens → Counter → P/R/F1 取所有 gt 中的最大 F1 适合长文本 caption anls (ANLS: Levenshtein 相似度) 1 - edit_distance / max_len threshold=0.5: 低于阈值为 0 TextVQA / DocVQA 用 contains gt in pred or pred in gt → 1.0 / 0.0 最宽松
# vqa.py 5种评分函数真实代码 def vqa_accuracy(prediction, ground_truths): norm_pred = normalize_answer(prediction) count = sum(1 for gt in ground_truths if normalize_answer(gt) == norm_pred) return min(count / 3.0, 1.0) def exact_match(prediction, ground_truths): norm = normalize_answer(prediction) return 1.0 if any(normalize_answer(gt)==norm for gt in ground_truths) else 0.0 def token_f1(prediction, ground_truths): pred_tokens = normalize_answer(prediction).split() best_f1 = 0.0 for gt in ground_truths: gt_tokens = normalize_answer(gt).split() common = Counter(pred_tokens) & Counter(gt_tokens) num_common = sum(common.values()) P = num_common / len(pred_tokens) R = num_common / len(gt_tokens) f1 = 2*P*R / (P+R) best_f1 = max(best_f1, f1) return best_f1 def anls_score(prediction, ground_truths, threshold=0.5): # Levenshtein edit distance dist = _levenshtein(norm_pred, norm_gt) nls = 1.0 - dist / max(len(pred), len(gt)) return max(nls, 0) if nls >= threshold else 0 def contains_match(prediction, ground_truths): norm_pred = normalize_answer(prediction) for gt in ground_truths: norm_gt = normalize_answer(gt) if norm_gt in norm_pred or norm_pred in norm_gt: return 1.0 return 0.0
